## 朝の5分で経営判断が終わる

毎朝やることは1つだけ。Claude Codeを開いて、こう打つ。

```
/ai-ceo:morning
```

これで、開発・マーケ・営業・経理・CS・法務の全6部門の状態が一覧で表示される。承認待ちのタスク、KPIの変動、今日やるべきことの推奨まで、すべてが5分で把握できる。

## 朝ダイジェストの中身

実際に表示される内容はこんなイメージだ。

```
📊 AI-CEO Morning Digest — 2026-03-26

## 承認待ち (3件)
- [AQ-019] マーケ: X投稿スレッド（書籍プロモ）
- [AQ-020] 営業: 取引先向けリリース告知メール
- [AQ-021] 営業: 知人向け近況報告メール

## 部門状態サマリ
| 部門 | ステータス | 進行中タスク |
|------|-----------|------------|
| 開発 | 🟢 正常   | LP改善完了  |
| マーケ | 🟢 正常 | SNS投稿自動配信中 |
| 営業 | 🟡 再編中 | コンサル営業にシフト |
| 経理 | 🟢 正常   | Q1決算準備中 |

## 今日のおすすめアクション
1. 承認待ち3件を処理（5分）
2. 書籍の販売数を確認（Zenn + Amazon）
3. Q1 KPIレビューの準備を開始
```

## 仕組みの裏側

朝ダイジェストは以下の3ステップで自動生成される。

### Step 1: 各部門のSTATE.mdを読み取る

```
.company/departments/
  ├── dev/STATE.md
  ├── marketing/STATE.md
  ├── sales/STATE.md
  ├── finance/STATE.md
  ├── cs/STATE.md
  └── legal/STATE.md
```

各部門のエージェントが前日までに更新した状態ファイルを自動収集。

### Step 2: 承認キューを確認

```
.company/approval-queue.md
```

対外アクションのドラフトが溜まっている承認キューから、未処理のアイテムを抽出。

### Step 3: KPIと推奨アクションを生成

過去のKPI推移と当日の状態を分析し、「今日やるべきこと」を優先順位付きで推奨。

## cronで完全自動化

毎朝の生成をcronで自動化すれば、起きたらSlackに通知が届いている状態になる。

```bash
# crontab
0 8 * * * /path/to/auto-morning.sh
```

スクリプトはダッシュボードAPIを叩いてダイジェストを生成し、Slack Webhookで通知する。

## CEOの判断は3つだけ

朝ダイジェストを見て、CEOが行う判断は3つだけ。

1. **承認する** — ドラフトを確認してOKなら承認
2. **差し戻す** — 修正が必要なら理由をつけて差し戻し
3. **優先度を変える** — 推奨アクションの順番を入れ替え

それ以外の実作業は、すべてAIエージェントが自動実行する。

## まとめ

朝の5分で全部門を把握し、3つの判断を下すだけ。残りの時間は、CEOにしかできない仕事（戦略判断、顧客対応、ネットワーキング）に使える。
