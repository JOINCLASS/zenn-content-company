## 数字を見ない経営は、目を閉じて運転するのと同じ

ひとり社長やフリーランスにありがちなのが「忙しくてKPIを見る時間がない」という状態。売上、コスト、リード数、コンバージョン率...管理すべき数字は多いが、集計する時間がない。

AIエージェントに任せれば、KPIの収集・集計・可視化が自動化できる。

## 追跡すべきKPI

事業フェーズによって追うべきKPIは異なるが、最低限これだけは押さえておきたい。

### 売上系
| KPI | 頻度 | データソース |
|-----|------|------------|
| 月間売上 | 月次 | freee / 銀行口座 |
| 書籍販売数 | 週次 | Zenn / Amazon KDP |
| 新規リード数 | 週次 | 問い合わせフォーム |
| コンバージョン率 | 月次 | GA4 |

### コスト系
| KPI | 頻度 | データソース |
|-----|------|------------|
| AI運用コスト | 月次 | Anthropic管理画面 |
| インフラコスト | 月次 | お名前.com / GCP |
| 広告費 | 週次 | Google Ads |

### マーケ系
| KPI | 頻度 | データソース |
|-----|------|------------|
| SNSフォロワー数 | 週次 | X / LinkedIn |
| 記事PV | 週次 | Zenn / GA4 |
| SEO順位（主要キーワード） | 月次 | Search Console |

## 自動追跡の仕組み

### STATE.mdによるKPI管理

```markdown
# .company/STATE.md

## KPIダッシュボード

### 書籍売上
| KPI | 現在値 | 目標 | 達成率 |
|-----|--------|------|--------|
| 販売部数 | 11部 | 50部 | 22% |
| 書籍売上 | ¥19,250 | ¥75,000 | 26% |
```

このファイルをAIエージェントが自動更新する。CEOが売上データを入力すれば、AIが達成率を計算し、トレンドを分析してくれる。

### 朝ダイジェストとの連携

毎朝の `/ai-ceo:morning` コマンドで、KPIの変動が自動レポートされる。

```
## KPI変動アラート
- 📈 書籍売上: 先週比 +3部（+¥6,000）
- ⚠️ AIコンサル無料診断: 今月0件（目標3件）
- 📉 SNSエンゲージメント率: 1.8%（目標2.0%）
```

## ダッシュボードの構築

より視覚的に管理したい場合は、Next.jsでダッシュボードを構築できる。

```typescript
// 部門状態を読み取ってJSON化
const departments = await parseDepartmentStates();
const kpis = await parseKPIs();

// APIエンドポイントで返す
app.get("/api/state", (req, res) => {
  res.json({ departments, kpis, lastUpdated: new Date() });
});
```

Markdownファイルをパースしてデータ化し、ブラウザで見やすく表示する。

## 意思決定への活用

KPIを追跡するだけでは意味がない。重要なのは **データに基づいて判断を変える** こと。

**判断例:**
- 書籍売上が伸びている → マーケ予算を書籍プロモに集中
- AIコンサルの無料診断が0件 → LP改善 or 集客チャネルの見直し
- AI運用コストが上限に近い → サブエージェント利用の最適化

## まとめ

KPI管理を自動化すれば、CEOは「数字を集める」作業から解放され、「数字を見て判断する」ことに集中できる。
