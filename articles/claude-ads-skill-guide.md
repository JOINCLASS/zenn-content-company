## 広告運用の監査、何時間かけていますか？

Google Ads、Meta Ads、LinkedIn Ads……。複数プラットフォームを横断した広告アカウントの監査は、手動でやると**4〜8時間**かかります。

- コンバージョントラッキングは正しく設定されているか？
- 入札戦略は最適か？
- クリエイティブの疲弊が起きていないか？
- 予算配分は適切か？
- コンプライアンスに違反していないか？

これらを全プラットフォームでチェックするのは、広告代理店でも一人の担当者では丸一日仕事です。

**Claude Ads**は、この作業を**5分以内**に完了する無料のオープンソーススキルです。

## Claude Adsとは

[Claude Ads](https://github.com/AgriciDaniel/claude-ads)は、Claude Code上で動作する広告監査・最適化スキルです。MITライセンスで無料公開されています。

### 対応プラットフォームとチェック項目数

| プラットフォーム | チェック項目数 | 主な監査内容 |
|----------------|-------------|-------------|
| Google Ads | 74項目 | 検索広告、PMax、Display、YouTube、Demand Gen |
| Meta Ads | 46項目 | Pixel/CAPI、クリエイティブ、オーディエンス、構造 |
| LinkedIn Ads | 25項目 | B2Bターゲティング、Lead Gen、TLA |
| TikTok Ads | 25項目 | クリエイティブ、Shop、Smart+ |
| Microsoft Ads | 20項目 | Google連携、Copilot統合 |
| クロスプラットフォーム | 63項目 | クリエイティブ品質、予算配分、コンプライアンス |

**合計190項目**を、6つの並列エージェントが同時にチェックします。

### アーキテクチャ

```
/ads audit（ユーザー入力）
  ↓
メインオーケストレーター
  ├── audit-google（Google Ads分析）
  ├── audit-meta（Meta Ads分析）
  ├── audit-creative（クリエイティブ品質）
  ├── audit-tracking（コンバージョン追跡）
  ├── audit-budget（予算・入札戦略）
  └── audit-compliance（コンプライアンス）
  ↓
統合レポート + Ads Health Score (0-100)
```

6つの専門エージェントが**並列で同時実行**されるため、全体の処理時間は最も遅いエージェントの完了時間に依存します。直列に実行するよりも大幅に高速です。

## Quality Gates（自動安全装置）

Claude Adsには、広告運用の「絶対に守るべきルール」がハードコードされています。

### 5つの必須ルール

**1. Broad Match + Smart Bidding ルール**
Google Adsでブロードマッチを使う場合、必ずスマート入札（目標CPA/ROAS）との併用が必要。手動入札でのブロードマッチは推奨しません。

**2. CPA 3倍ルール**
目標CPAの3倍を超えた広告グループ/キャンペーンには即座に停止フラグが立ちます。

```
目標CPA: ¥3,000 → ¥9,000超でアラート
```

**3. 最低予算チェック**

| プラットフォーム | 最低予算 |
|----------------|---------|
| Meta Ads | CPA × 5倍 / 広告セット |
| TikTok Ads | CPA × 50倍 / 広告グループ |

予算不足の広告セットは学習が進まず、パフォーマンスが安定しません。

**4. 学習フェーズの保護**
広告が学習フェーズ中（通常50コンバージョン到達まで）は、編集を推奨しません。学習リセットはパフォーマンス低下の最大原因です。

**5. コンプライアンスチェック**
住宅・雇用・金融に関する広告は「特別広告カテゴリ」の宣言が必要。宣言なしでの配信はポリシー違反になります。

## Ads Health Score の読み方

`/ads audit` 実行後、0-100のスコアで総合評価が出ます。

| スコア | グレード | 意味 | 対応 |
|--------|---------|------|------|
| 90-100 | A | 微調整のみ | 月次メンテナンスを継続 |
| 75-89 | B | 改善余地あり | Quick Winsから着手 |
| 60-74 | C | 要対応の問題あり | 週次で改善サイクルを回す |
| 40-59 | D | 重大な問題あり | 即座に対応計画を策定 |
| 0-39 | F | 緊急介入が必要 | 広告を一時停止して根本的に見直す |

スコアは重み付きで計算されます。コンバージョントラッキングの問題は重大（高ウェイト）、クリエイティブの小さな改善は軽微（低ウェイト）として評価されるため、実態に即した評価になります。

## AI経営フレームワーク（AI-CEO）との連携

筆者は、CEO1名 + AIエージェントで会社を経営する「AI-CEO Framework」を運用しています。Claude Adsはこのフレームワークに統合し、以下のように活用しています。

### 統合コマンド

```
/ai-ceo:mkt:ads-audit    → Claude Adsの/ads auditを実行
/ai-ceo:mkt:ads-plan     → 業界別広告戦略テンプレート生成
```

### 活用フロー

```
1. 月初: /ads audit で全プラットフォーム監査
2. スコア確認: Health Score + Quick Wins抽出
3. 改善実行: 指摘事項を優先度順に対応
4. 月末: 再監査でスコア変化を確認
```

### 他部門との連携

| 連携先 | 連携内容 |
|--------|---------|
| 営業部門 | 広告経由のリード品質をフィードバック → ターゲティング改善 |
| 経理部門 | 広告費の勘定科目自動分類（広告宣伝費） |
| 開発部門 | LP品質の `/ads landing` 結果 → コンバージョン改善 |
| コンサル部門 | 顧客の広告アカウントを `/ads audit` で診断 → サービス提供 |

特にコンサルティング事業との相性が良く、**顧客の広告アカウントを5分で診断 → 改善提案**というサービスフローが実現できます。

## 中小企業での活用ポイント

### 月3万円からの広告運用

中小企業がClaude Adsを活用する最適なステップ：

1. **`/ads plan <業種>`** で戦略テンプレートを生成
2. Google Adsを日予算1,000円で開始（月3万円）
3. 2週間後に **`/ads google`** で最初の監査
4. Quick Winsを実行して改善
5. 効果が出たら予算を段階的に増額

### 広告代理店に依頼する前に

月額数十万円の広告代理店に依頼する前に、Claude Adsで自社監査してみてください。

- 代理店の提案が妥当かどうかのセカンドオピニオンとして使える
- 自社で基礎的な運用をした上で、代理店に高度な運用を任せる判断ができる
- 代理店からのレポートの読み解き精度が上がる
