## 1. 設定管理

### Permissions
- [ ] Enterprise Policy / Organization settingsが最新か
- [ ] denyリストに必須項目（.env, force push, rm -rf, sudo）が含まれるか
- [ ] ロール別の設定が適切に適用されているか
- [ ] 設定変更のCI検知が機能しているか

### CLAUDE.md
- [ ] 全プロジェクトにセキュリティポリシーを含むCLAUDE.mdがあるか
- [ ] CLAUDE.mdの内容が最新のポリシーと整合しているか
- [ ] 変更履歴が適切に管理されているか

### Hooks
- [ ] 監査ログHookが全プロジェクトで有効か
- [ ] 機密情報フィルタリングHookが正常に動作するか
- [ ] Hookスクリプトのテストが定期的に実施されているか

## 2. ログ分析

### 監査ログ
- [ ] ログが継続的に記録されているか
- [ ] ログの保持期間が規定に準拠しているか
- [ ] 異常な利用パターンがないか
  - 大量のファイル読み取り
  - 深夜・休日の利用
  - ブロックされた操作の頻発

### ブロックされた操作
- [ ] ブロック理由の分析が行われているか
- [ ] 意図的な回避の試みがないか
- [ ] ブロックルールの過不足がないか

## 3. アクセス管理

- [ ] 退職者のアカウントが無効化されているか
- [ ] ロール変更に伴うPermissionsの更新が行われているか
- [ ] 不要なアカウントが存在しないか

## 4. ネットワーク

- [ ] プロキシサーバーが正常に動作しているか
- [ ] コンテンツフィルタリングが機能しているか
- [ ] ファイアウォールルールが最新か
- [ ] 不正な外部通信がないか

## 5. シークレット管理

- [ ] git-secrets / detect-secretsが全リポジトリで有効か
- [ ] pre-commitフックが機能しているか
- [ ] シークレットのローテーションが実施されているか
- [ ] .envファイルがgitにコミットされていないか

## 6. コンプライアンス

- [ ] DPA/利用規約が最新か
- [ ] プライバシーポリシーの記載が最新か
- [ ] 利用者の同意書が取得されているか
- [ ] 教育・研修が実施されているか

## 7. インシデント対応

- [ ] インシデント対応手順が最新か
- [ ] 連絡先一覧が最新か
- [ ] インシデント訓練が定期的に実施されているか
