# CSO部門の自動化 -- 提案書・リード管理

## エンジニアにとって営業は「異世界」だった

開発、マーケティング、経理。ここまでの3部門は、少なくとも業務内容のイメージがついた。しかし営業は違う。

「見込み客にアプローチする」「ヒアリングして提案書を書く」「価格交渉をして受注する」。言葉は知っているが、エンジニアとして10年以上コードを書いてきた人間にとって、これらは完全に異世界の作法である。

合同会社ジョインクラスのDX支援事業は、これまで紹介ベースで案件を獲得していた。知人から「こういうシステムを作ってほしい」と依頼が来て、見積もりを出して、開発する。このフローに「営業」は存在しなかった。

しかし、紹介だけに頼っていては事業を拡大できない。Q1末の目標は月間売上¥345,000だが、新規リード数はゼロの状態だった。ここでCSOエージェント（営業部長）の出番である。

## CSOエージェントの役割定義

CSOエージェントは、以下のペルソナで定義している。

```markdown
# CSO / 営業部長エージェント

あなたは AI-CEO Framework の CSO（最高営業責任者）です。

## ペルソナ

ソリューション営業のスペシャリスト。SaaS・B2Bの営業プロセスに精通し、
PLG（Product-Led Growth）とSLG（Sales-Led Growth）の両方を使い分ける。
フリーランスや中小企業向けの提案に特に強い。
```

ポイントは、PLGとSLGの両方を扱える点である。Focalizeのようなセルフサービス型SaaSにはPLG（プロダクト主導のグロース）が適しているが、DX支援やAIコンサルのような受託型サービスにはSLG（営業主導のグロース）が必要になる。6つのプロダクトと2つのサービスを抱える当社では、この使い分けが不可欠である。

### CSOエージェントの専門知識

```markdown
## 専門知識

### 営業戦略
- PLG戦略: 無料プラン→有料転換の最適化
- SLG戦略: DX支援・コンサル案件の対面営業プロセス
- ABM（Account-Based Marketing）: ターゲット企業へのアプローチ手法

### パイプライン管理
- ステージ設計: リード→初回接点→商談→提案→交渉→受注/失注
- 売上予測: ステージ別受注確度と加重パイプライン計算
- ボトルネック分析: 各ステージの滞留日数と離脱率

### 提案・見積
- SaaS提案書のフレームワーク: 課題→解決策→ROI→導入計画→価格
- DX支援の見積方法: 工数×単価、成果報酬型、リテイナー型
- 競合比較資料の作成
```

これらのテンプレートは`.company/departments/sales/proposals/`に格納される。新しい商談が発生した際、CSOエージェントがテンプレートをベースにカスタマイズして提案書ドラフトを生成する。

### 提案書生成のワークフロー

```markdown
### /ai-ceo:sales:proposal "対象" -- 提案書生成
1. 対象企業/顧客のニーズを整理
2. マッチするプロダクト/サービスを選定
3. 提案書ドラフトを作成:
   - 課題の明確化
   - 解決策（自社プロダクト/サービス）
   - 導入効果（定量的ROI）
   - 導入スケジュール
   - 料金プラン
4. .company/departments/sales/proposals/ に出力
5. approval-queue に追加（顧客送付前にCEO確認）
```

提案書は「常にdraft」として扱われる。permissions.mdでの分類は以下のとおり。

```yaml
always_draft:
  - proposal    # 提案書は常にCEO承認が必要
  - contract
  - invoice
```

対外的に送付する提案書をAIに全自動で送信させることは絶対にしない。金額の妥当性、技術的な実現可能性、クライアントとの関係性の文脈を、CEOが最終確認する必要がある。

## リード獲得戦略の自動策定

### 現状認識 -- リード数ゼロからのスタート

CSOエージェントに最初に依頼したのは、リード獲得戦略の策定だった。現状は以下のとおり。

```markdown
## 1. 現状認識

| 指標 | 現在値 | Q1末目標 | ギャップ |
|------|--------|----------|----------|
| DX支援 新規リード数/月 | 0 | 3 | 3件 |
| DX支援 月間売上 | 0 | 300,000 | 300,000 |
| コンサル 無料診断実施数 | 0 | 3 | 3件 |
| コンサル 受注件数 | 0 | 1 | 1件 |
| コンサル MRR | 0 | 100,000 | 100,000 |

**残り期間:** 約5週間
**最重要課題:** リード数がゼロ。まず「打席に立つ」ことが最優先。
```

「まず打席に立つことが最優先」。この一言が、CSOエージェントの本質を表している。完璧な営業戦略を練るより先に、まず1件でもアプローチすることが重要だという判断を、AIが自ら下した。

### ターゲット企業像（ICP）の定義

CSOエージェントは、2つのサービスそれぞれに対してICP（Ideal Customer Profile）を定義した。

```markdown
### DX支援/受託開発
| 属性 | 条件 |
|------|------|
| 業種 | 教育、物流、飲食、小売、製造業 |
| 企業規模 | 従業員10-200名の中小企業 |
| 予算感 | 月額30-100万円 |
| 決裁者 | 経営者/役員クラス |

**ペインポイント:**
1. 業務がExcel/紙に依存、属人化・ミスが頻発
2. 大手SIerは高すぎる/小規模案件を受けてくれない
3. 社内にエンジニアがおらず、ITの相談相手がいない

### AI業務自動化コンサルティング
| 属性 | 条件 |
|------|------|
| 業種 | 教育（塾、スクール）、物流（配送、倉庫）、飲食（チェーン、FC） |
| 企業規模 | 従業員20-300名 |
| 予算感 | 月額10-30万円、初期15-50万円 |
| 導入動機 | 「AI活用に興味はあるが、何から始めればいいかわからない」 |
```

ペインポイントの3番目「社内にエンジニアがおらず、ITの相談相手がいない」は、当社の差別化ポイントに直結する。大手SIerや大規模なコンサル会社は、こうした小規模企業の相談に乗らない。だからこそ、少人数で柔軟に対応できる合同会社としての強みが活きる。

### チャネル別アクション計画

リード獲得のチャネルは4つに分類した。

```markdown
### 3-1. インバウンド
| アクション | 期限 | KPI |
|-----------|------|-----|
| DX支援サービスページ追加 | 3/1 | ページ公開 |
| AI自動化コンサルのサービスページ追加 | 3/1 | ページ公開 |
| 無料AI業務診断の申し込みフォーム設置 | 3/1 | フォーム稼働 |
| 各記事末尾にCTA設置 | 記事公開時 | CTAクリック率 2%+ |

### 3-2. アウトバウンド
| アクション | 期限 | KPI |
|-----------|------|-----|
| ターゲット企業リスト30社の作成 | 3/3 | リスト完成 |
| LinkedInプロフィール最適化 | 3/1 | プロフィール更新 |
| コールドメール送信（週10通） | 毎週 | 返信率 5%+ |

### 3-3. リファラル（最速チャネル）
| アクション | 期限 | KPI |
|-----------|------|-----|
| 過去の取引先・知人リストの棚卸し | 2/28 | リスト完成 |
| 紹介依頼メール送信（10通） | 3/5 | 10通送信 |
| 紹介特典の設計（成約で10%バック） | 3/7 | 設計完了 |

### 3-4. パートナーシップ
| アクション | 期限 | KPI |
|-----------|------|-----|
| 税理士/社労士事務所への提携提案（3社） | 3/10 | 提案3件 |
| Webデザイン会社への共同提案パートナー打診（2社） | 3/10 | 提案2件 |
```

注目すべきは、「リファラル（紹介）」を「最速チャネル」と位置付けている点である。コールドメールの返信率が5%前後であるのに対し、紹介経由の商談化率ははるかに高い。1人法人が限られたリソースで最大の成果を出すには、まず既存の人間関係を活用するのが合理的である。

## アウトリーチメールの自動生成と品質管理

### 4種類のメールテンプレート

CSOエージェントは、4種類のアウトリーチメールテンプレートを生成した。

**テンプレート1: コールドメール -- DX支援向け**

```
件名: {company_name}様の業務効率化について、1つご提案があります

{recipient_name} 様

突然のご連絡を失礼いたします。
合同会社ジョインクラスの{ceo_name}と申します。

{company_name}様の{specific_observation}を拝見し、
業務のシステム化でお力になれるのではないかと思い、ご連絡いたしました。

当社は中小企業様向けに、DX推進支援（業務システム開発・Webアプリ開発）を
行っております。特に以下のようなお悩みをお持ちの企業様をご支援しています。

- Excel や紙ベースの業務が多く、二重入力やミスが発生している
- 外注先を探しているが、大手は予算に合わない
- 社内にエンジニアがおらず、ITの相談相手がいない

30分のオンラインミーティングで、貴社の状況に合った改善策を
無料でご提案いたします。
{calendar_link}
```

**テンプレート2: コールドメール -- AI自動化コンサル向け**

```
件名: {industry}業界のAI活用事例と無料診断のご案内

{recipient_name} 様

はじめまして。合同会社ジョインクラスの{ceo_name}です。
{industry}業界におけるAI活用支援を専門に行っております。

**無料のAI業務診断（30分）**を実施中です。

診断でわかること:
1. AI自動化が有効な業務領域の特定
2. 工数削減効果の概算（時間・金額）
3. 段階的な導入ステップの提案
```

テンプレートには `{specific_observation}` のような変数が埋め込まれている。CSOエージェントが送信先の企業ごとにカスタマイズする部分である。このパーソナライゼーションが、コールドメールの返信率を左右する。

### 業界別の差し込み文

AI自動化コンサル向けのメールには、業界別の差し込み文も用意した。

```markdown
### 業界別差し込み文

**教育:**
保護者対応AIチャットボット（対応工数最大90%削減）、
成績レポート自動生成（月20h削減）

**物流:**
FAX/メール受注のAI読取り→自動登録（入力ミスゼロ）、
配送状況FAQ自動応答（80%削減）

**飲食:**
LINE/電話の予約AI自動受付（80%削減）、
口コミ自動収集・分析・返信案生成
```

具体的な数字（「90%削減」「月20h削減」「入力ミスゼロ」）が含まれている点が重要である。抽象的な「効率化のお手伝いをします」ではなく、具体的なベネフィットを示すことで、受信者の関心を引く。

### メール送信の品質管理

すべてのアウトリーチメールは、permissions.mdによって「常にdraft」に分類される。

```yaml
always_draft:
  - client_communication  # クライアントへの連絡は常にCEO確認
```

CSOエージェントがテンプレートから生成したメールは、approval-queueに追加され、CEOが1通ずつ確認してから送信する。なぜなら、メールは会社の「顔」であり、1通のミスが信用を損なう可能性があるからである。

### フォローアップルール

CSOエージェントは、フォローアップのルールも定義した。

```markdown
### フォローアップルール
| 回数 | タイミング | 内容 |
|------|-----------|------|
| 1回目 | 3営業日後 | 簡潔にリマインド |
| 2回目 | 1回目から5営業日後 | 新情報を添えて再アプローチ |
| 3回目 | 2回目から7営業日後 | 最後のご連絡 |

3回フォローしても反応なしの場合 → 3ヶ月後に再アプローチリストへ
```

「3回で止める」というルールは、しつこい営業にならないためのガードレールである。AIに任せると無限にフォローアップメールを送り続けるリスクがあるため、明示的に上限を設定している。

## パイプライン管理の仕組み

### 6ステージのパイプライン設計

CSOエージェントは、営業パイプラインを6ステージで設計した。

```markdown
### パイプライン管理
- ステージ設計: リード→初回接触→商談→提案→交渉→受注/失注
- 売上予測: ステージ別受注確度と加重パイプライン計算
- ボトルネック分析: 各ステージの滞留日数と離脱率
```

各ステージの目標転換率は以下のとおり。

```markdown
## 営業パイプライン目標転換率

| ステージ | 目標転換率 | 必要数（逆算） |
|---------|-----------|--------------|
| リード | - | 月10件 |
| 初回接触 | 50% | 月5件 |
| ヒアリング | 60% | 月3件 |
| 提案 | 70% | 月2件 |
| 受注 | 50% | 月1件 |
```

月1件の受注を達成するためには、月10件のリードが必要。逆算してアクション量を決定する。これは営業の基本だが、エンジニア出身者にとっては目から鱗の考え方である。

### Focalizeとの連携

パイプライン管理のツールとしては、自社プロダクトのFocalizeを活用する設計にした。

```markdown
### CRM/ツール
- Focalize（自社ツール）を活用した営業プロセス管理
- パイプラインダッシュボードの設計
```

Focalizeは、メール転送だけで商談情報を自動管理するツールである。営業メールをFocalizeの専用アドレスに転送すれば、AIが内容を解析して商談ステージを自動分類する。自社の営業にも自社ツールを使うことで、プロダクトの改善点を自ら体験する「ドッグフーディング」にもなる。

## 実践例: 5週間のリード獲得アクションカレンダー

CSOエージェントが策定した5週間のアクションカレンダーの全容を示す。

### Week 1（2/24-28）-- 準備フェーズ
```
- 過去の取引先・知人リストの棚卸し
- LinkedInプロフィール最適化
```

まず「誰にアプローチできるか」を棚卸しする。新規開拓の前に、既存の人間関係を活用する。

### Week 2（3/1-7）-- 基盤構築フェーズ
```
- サービスページ公開、フォーム設置
- ターゲット企業リスト30社作成
- 紹介依頼メール10通送信
```

この週が最も重要である。サービスページの公開とフォーム設置により、リードの受け皿を整備する。同時に、紹介依頼メールを10通送信する。紹介はリファラルチャネルの核であり、最速でリードを獲得できる手段である。

### Week 3（3/8-14）-- アウトバウンド開始
```
- コールドメール送信開始（週10通）
- 提携提案（税理士/社労士、Webデザイン会社）
```

Week 2で基盤ができたら、アウトバウンドを開始する。コールドメールは週10通からスタートし、返信率を見ながら調整する。

税理士や社労士への提携提案は、CSOエージェントならではの視点だった。税理士は中小企業の経営者と日常的に接点があり、「システム化したい」という相談を受けることがある。そこで当社を紹介してもらう仕組みを作る。

### Week 4（3/15-21）-- 改善フェーズ
```
- コールドメール第2バッチ、返信率分析
- セミナー/勉強会登壇の打診
```

第2バッチでは、Week 3の返信率を分析して件名やメール本文を改善する。A/Bテストの発想を営業メールにも適用する。

### Week 5（3/22-31）-- 振り返りフェーズ
```
- パイプライン棚卸し、Q2計画反映
- 月次KPIレビュー
```

5週間の結果を棚卸しし、何が効いて何が効かなかったかを分析する。この結果をQ2の営業計画に反映する。

## CSO部門の権限設計

```markdown
## 権限レベル

- **execute:** パイプライン分析、提案書ドラフト、競合分析、営業資料作成
- **draft:** 顧客への提案送付、価格交渉、契約条件の提示
```

ここでも「分析・作成は自動実行、対外アクションはCEO承認」という原則が貫かれている。

特に重要なのは「価格交渉」と「契約条件の提示」がdraftに分類されている点である。価格の設定ミスは直接的な収益損失につながる。AIが自動で値引きを提案するような事態は避けなければならない。

## CSO部門の品質検証

```markdown
## 品質検証

1. **顧客視点:** 提案が顧客の課題を正確に捉えているか
2. **ROI明確性:** 導入効果が定量的に示されているか
3. **実現可能性:** 提案内容が技術的・リソース的に実現可能か
4. **価格整合:** pricing 設定と矛盾していないか
```

3番の「実現可能性」は特に注意が必要である。AIは「理想的な提案」を書くのが得意だが、実際にCEO1名で開発・導入・サポートできるかどうかは別の話である。提案書に書いた内容を実行できなければ、信用を失う。そのため、技術的・リソース的な制約を考慮した上でCEOが最終判断する。

## まとめ -- AIが営業の壁を下げた

CSO部門の自動化により、以下のことが実現した。

**Before（自動化前）:**
- 営業活動は紹介ベースのみ
- 提案書は案件ごとにゼロから作成
- リード獲得の戦略が存在しない
- パイプライン管理の仕組みがない
- 新規リード数: ゼロ

**After（自動化後）:**
- 4種類のアウトリーチメールテンプレートが完成
- 2種類の提案書テンプレートが完成
- 5週間のアクションカレンダーが策定済み
- 4チャネル（インバウンド、アウトバウンド、リファラル、パートナーシップ）の計画が明確化
- パイプライン管理の6ステージが設計済み

もちろん、テンプレートとカレンダーが完成しただけでは受注には至らない。最終的にはCEOが自分の声でクライアントと話し、信頼関係を築く必要がある。しかし、「何をすればいいかわからない」という状態から「やるべきことが明確になった」状態への変化は、行動を起こす心理的ハードルを大きく下げる。

次章では、ここまで各部門で繰り返し触れてきた「承認パイプライン」の全体像を解説する。AIに全権委任してはいけない理由と、draft/承認/実行の3段階で品質とガバナンスを担保する仕組みについて、具体的な実例を交えて詳しく述べる。
