## ツール選びで失敗しないたった1つのルール

ここまでの章で、日次レポート、問い合わせ対応、経理の3つの業務を自動化する方法を解説してきました。

この章では「どのツールを使うか」を決める方法を解説します。

ツール選定で最も重要なルールは1つだけです。

**「今の月間売上の5%以内に収める」**

月商¥500,000の会社なら、ツール代は月¥25,000まで。月商¥1,000,000なら月¥50,000まで。

なぜ5%なのか。それ以上かけると、自動化のコスト削減効果を自動化ツール自体のコストが食ってしまうからです。

私のクライアントで、月商¥800,000の会社が月¥80,000のツールを契約していたケースがありました。売上の10%がツール代に消えている。自動化で月20時間の削減ができても、時給換算¥4,000で¥80,000。トントンです。つまり自動化の意味がない。

ツール代を月¥10,000に抑えて同じ20時間を削減できれば、月¥70,000の実質的な利益です。

**安いツールで始める。効果が出たら投資を増やす。** この原則を守ってください。


## 著者の全ツールスタック公開

私の会社で使っている全ツールと月額コストを公開します。

### メインツール

**Claude Code Max — 約¥30,000/月**

これが自動化の心臓部です。すべての自動化タスクのエンジンとして使っています。レポート生成、問い合わせ分類、仕訳作成、コード生成、文書作成、データ分析。1つのツールでこれだけの業務をカバーできるのが最大の魅力です。

1Mトークン（約75万文字）のコンテキストウィンドウにより、過去1年分のデータを一度に読み込んで分析できます。「過去のパターンを踏まえて、この取引をどう仕訳すべきか」という判断が正確にできるのは、この大きなコンテキストがあるからです。

**launchd — ¥0/月（macOS標準機能）**

17個のジョブを動かしています。朝のレポート生成、SNS投稿のスケジュール実行、データのバックアップ、Webサイトの監視など。macOSを使っていれば追加コストゼロです。

Windowsの場合は「タスクスケジューラ」が同等の機能を提供します。こちらも無料です。

### 無料で使っているツール

**Googleスプレッドシート — ¥0**

売上データ、顧客リスト、KPI管理などのマスターデータはすべてスプレッドシートで管理しています。Google Apps Scriptで自動化処理も書けます。

**GitHub — ¥0（無料プラン）**

コード管理だけでなく、社内の文書管理にも使っています。変更履歴が自動で残るので、「いつ誰が何を変えたか」が追跡できます。GitHub Actionsの無料枠（月2,000分）で定期実行タスクも動かせます。

**Slack — ¥0（無料プラン）**

社内コミュニケーションとbot通知の受け口として使っています。無料プランでもIncoming Webhookが使えるので、自動通知には十分です。

### 少額の有料サービス

**ドメイン・サーバー代 — 約¥2,280/月**

Webサイトのホスティングとドメイン維持費です。

### 合計: 約¥32,280/月

この金額で、会社の業務の98%を自動化しています。

残りの2%は何かというと、クライアントとの商談、契約書の最終確認、経営の意思決定。つまり「人間がやるべき業務」です。


## Claude Code Opus 4.7の最新機能と実務活用

Claude Codeは急速に進化しています。2025年時点の最新機能と、それぞれの実務での活用方法を解説します。

### Remote Tasks — スケジュール実行

先ほど経理の章でも触れましたが、改めて詳しく説明します。

Remote Tasksは、クラウド上でClaude Codeのタスクを定期実行できる機能です。自分のパソコンが起動していなくても、指定した時間にタスクが実行されます。

**実務での活用例:**
- 毎朝7:00に売上レポートを生成してSlackに投稿
- 毎月末に仕訳の自動生成と整合性チェック
- 毎週月曜に先週のKPIサマリーを作成

launchdやcronとの違いは、「自分のPCを起動しておく必要がない」こと。出張中でも、休暇中でも、タスクが確実に実行されます。

### Subagents — 専門タスクの委任

Subagentsは、Claude Codeの中で「専門家のAI」を別途起動して、特定のタスクを委任する機能です。

例えば、月末の経理処理を実行するとき、メインのClaude Codeが「経理専門のSubagent」を起動します。このSubagentには、経理に特化した知識（過去の仕訳パターン、勘定科目のルール、税務の基本知識）が事前に設定されています。

**実務での活用例:**
- 経理Subagent: 仕訳の自動生成に特化
- 営業Subagent: 提案書のドラフト作成に特化
- レポートSubagent: データ分析とレポート生成に特化

私の会社では11の「部門」に分かれたSubagentを運用しています。経営管理、マーケティング、営業、法務、経理、技術など。各Subagentが自分の専門分野のタスクを処理し、メインのClaude Codeが全体を統括しています。

### Agent Teams — 複数エージェントの協調作業

Agent Teamsは、複数のSubagentが連携して1つのプロジェクトを進める機能です。

例えば「新サービスのランディングページを作る」というタスクでは、コピーライティングのエージェント、デザインのエージェント、コーディングのエージェントが協調して作業します。

これは上級者向けの機能ですが、将来的に自動化を拡大する際に強力なツールになります。

### Hooks — 品質ゲート

Hooksは、特定のアクションの前後に自動チェックを入れる機能です。

**実務での活用例:**
- 仕訳を確定する前に、金額の整合性を自動チェック
- メールを送信する前に、個人情報が含まれていないか自動スキャン
- コードをデプロイする前に、テストが全て通っているか確認

第4章で解説した「承認パイプライン」を、技術的に実装するのがHooksです。

### MCPサーバー — 外部システム連携

MCPサーバーについては第4章で触れましたが、ここで全体像をまとめます。

MCPサーバーは、Claude Codeが外部のサービス（メール、CRM、会計ソフト、Slack、データベース等）と直接やり取りするための仕組みです。

MCPサーバーなしの場合、データの受け渡しは手動でコピー＆ペーストする必要があります。MCPサーバーありの場合、Claude Codeが必要なデータを自動で取得し、処理結果を自動で書き戻します。

設定は少し手間がかかりますが、一度設定すれば、以降は完全自動で連携できます。

### Background Agents + Git Worktree — 並列作業

複数のタスクを同時に処理したい場合、Background Agentsを使います。

通常のClaude Codeは1つのタスクを順番に処理しますが、Background Agentsを使えば、レポートの生成をしながら仕訳の処理をする、といった並列作業が可能になります。

Git Worktreeは、同じプロジェクトの異なるブランチを同時に操作するための仕組みです。技術的な開発タスクで特に有用です。


## 無料ツールだけで始める場合のスタック

「Claude Code Maxの¥30,000は今は出せない」という方向けの、完全無料のツールスタックを提案します。

### ¥0/月のスタック

**Claude（claude.aiの無料プラン）**

コードの生成、文書の作成、データの分析に使います。無料プランは利用量に制限がありますが、中小企業の日常的なタスクには十分です。

**Googleスプレッドシート + Google Apps Script**

データ管理と自動化の基盤です。スプレッドシートでデータを管理し、Apps Scriptで集計・通知を自動化します。

Apps Scriptでできること:
- スプレッドシートの更新を検知して自動でメール通知
- 毎朝の売上集計を自動実行
- Googleフォームの回答を自動で分類

**Gmail**

レポートの自動送信に使います。Apps Scriptから直接Gmailを送信できるので、追加のツールは不要です。

**GitHub（無料プラン）**

コード管理と文書管理。GitHub Actionsの無料枠で定期実行も可能です。

### 無料スタックで実現できること

この構成で、以下の自動化が可能です。

- 毎朝の売上レポートのメール配信（第3章の内容）
- 問い合わせの手動分類と回答テンプレートの管理（第4章の簡易版）
- 月次の経費集計と推移表の自動生成（第5章の簡易版）

「問い合わせのAI自動分類」はClaude無料プランの制限内では難しいですが、分類ルールをスプレッドシートに定義して、Apps Scriptでキーワードベースの自動分類を行うことは可能です。AI精度には劣りますが、手動よりは格段に楽です。

## ツール選定で陥りがちな罠

### 罠1: 機能が多いツールを選んでしまう

「このツールは100個の機能がある」と「このツールは必要な10個の機能だけある」なら、後者を選ぶべきです。

使わない90個の機能のために高い料金を払う必要はありません。しかも、機能が多いツールほど使い方が複雑で、学習コストが高くなります。

私の経験では、高機能ツールの全機能のうち、実際に使うのは5〜20%です。残りの80〜95%は「あったら便利かも」レベルで、実際には使いません。

### 罠2: 連携のためにツールを増やしてしまう

「AツールとBツールを連携するためにCツールを入れて、Cツールの管理のためにDツールを...」

ツールが増えれば増えるほど、管理コストが上がります。連携の不具合も増えます。1つのツールが障害を起こすと、連鎖的に他のツールも止まります。

**ツールの数は、少なければ少ないほど良い。**

私の会社のメインツールはClaude Code Maxの1つだけ。これに無料ツール（スプレッドシート、GitHub、Slack）を組み合わせています。有料ツールが1つだけなので、管理が非常に楽です。

### 罠3: 「話題のツール」に飛びつく

AIツールの世界は変化が速く、毎月のように「革新的な新ツール」が登場します。

新しいツールが出るたびに乗り換えていたら、移行コストだけで膨大な時間が消えます。私のルールは「今のツールで本当にできないことがあるか？」を確認してから、新ツールを検討することです。

「できないこと」ではなく「やり方を知らないだけ」ということが大半です。

### 罠4: 年間契約で安くなるからと長期契約する

月額¥10,000のツールが年間契約で20%オフ、¥96,000。お得に見えます。

でも、3ヶ月で使わなくなったら？ ¥96,000が丸損です。月額¥10,000 × 3ヶ月 = ¥30,000で済んだはずが、3倍以上の損失です。

**新しいツールは、最低3ヶ月は月額契約で使う。** 3ヶ月使って確実に継続すると判断してから、年間契約を検討してください。


## ツール導入の正しい手順

最後に、新しいツールを導入する際の正しい手順をまとめます。

### ステップ1: 「何を解決したいか」を明確にする

ツールから探すのではなく、課題から探します。「毎朝のレポート作成が大変」が課題なら、その課題を解決できるツールを探す。「最近AIツールが話題だから」でツールを探さない。

### ステップ2: 無料で試せるか確認する

ほとんどのツールには無料プランか無料トライアルがあります。まず無料で試して、「自社の業務に合うか」を確認します。

### ステップ3: 1つの業務で小さく始める

いきなり全社導入しない。まず1つの業務（例: 売上レポートの自動化）だけにツールを使う。効果を確認してから、対象業務を広げる。

### ステップ4: 3ヶ月使って評価する

1週間で判断しない。ツールの真価は、3ヶ月使い続けて初めてわかります。初期設定の手間、慣れるまでの時間、チューニングの効果、すべてを含めて3ヶ月後に「投資対効果があるか」を判断する。

### ステップ5: 不要になったらすぐ解約する

「もったいないから」で使わないツールを契約し続けるのが、最ももったいないです。使っていないツールは即解約。


## この章のまとめ

ツール選定の鉄則は「売上の5%以内」「少ないほど良い」「無料から始める」の3つ。

- 月¥0でも、Googleスプレッドシート + Apps Scriptで基本的な自動化は可能
- 月¥5,000で、AI分類やSlack通知を追加できる
- 月¥30,000で、著者と同じ98%自動化が実現できる
- Claude Codeは「1つのツールで複数業務をカバーできる」のが最大の強み

次章からは、これらのツールを使って「組織全体に自動化を広げる」方法を解説します。最初の3つの業務を自動化した成功体験を、どうやって全社に波及させるかがテーマです。


## この章のアクションリスト

1. 現在契約しているすべてのツール・サービスの月額料金を一覧にしてください。合計額は月商の何%ですか？
