## なぜ経理の自動化は3番目なのか

ここまでの流れを振り返ります。

第3章では「レポート」を自動化しました。データを見るだけで、何も書き換えない、最もリスクの低い自動化でした。

第4章では「問い合わせ対応」を自動化しました。AIが判断して下書きを作り、人間が確認して送信する。「承認パイプライン」の考え方を学びました。

そして今章の「経理の自動化」。ここで初めて、お金に直接関わる業務の自動化に踏み込みます。

経理の自動化を3番目にした理由は明確です。

**お金を扱う業務は、間違えると実害がある。**

仕訳を間違えれば税務申告に影響します。請求書の金額を間違えればクライアントの信用を失います。経費の分類を間違えれば、税務調査で問題になる可能性があります。

だからこそ、レポート自動化と問い合わせ自動化で「自動化の仕組み」「承認パイプラインの運用」「AIのフィードバック方法」を学んでから取り組むべきなのです。

ただし、正しく実装すれば、経理の自動化は3つの中で最も投資対効果が高いです。


## 著者の実装例: freee × Claude連携で月末経理を半日→30分に短縮

私の会社の月末経理は、以前はこうでした。

**Before（手動時代）:**
1. freeeにログインして、銀行口座の明細を取り込む（10分）
2. 取り込んだ明細を1件ずつ仕訳する（2時間）
3. クレジットカードの明細を確認して仕訳する（1時間）
4. 経費のレシートを整理して入力する（30分）
5. 売上の計上漏れがないか確認する（30分）
6. 仕訳の整合性チェック（残高が合うか確認）（30分）
7. 月次推移表を作成して前月と比較する（30分）
8. 必要な帳票を出力して保存する（15分）

合計: 約5.5時間（半日）

毎月この作業を月末の2〜3日に集中して行っていました。月末は他の業務も忙しいので、経理作業は深夜に回すことも多かったです。

**After（自動化後）:**
1. Claude Codeが銀行明細を自動取得して仕訳を自動生成（自動、5分で完了）
2. クレジットカード明細も同様に自動仕訳（自動）
3. 経費レシートはスマホで写真を撮って送るだけ。AIが内容を読み取って仕訳（1件1分）
4. 売上の計上漏れチェックを自動実行、怪しい取引をリストアップ（自動）
5. AIが生成した仕訳を一覧で確認、問題がなければ承認（20分）
6. 月次推移表も自動生成（自動）
7. 帳票の出力・保存も自動（自動）

合計: 約30分（仕訳の確認作業のみ）

**削減効果: 5.5時間 → 30分（91%削減）**

年間に換算すると、5時間 × 12ヶ月 = 60時間の削減です。時給¥5,000で計算すると、年間¥300,000の価値があります。


## 自動化できる経理業務の一覧

「経理の自動化」と一口に言っても、すべてを一度に自動化する必要はありません。自動化しやすい業務と、しにくい業務があります。

### 自動化しやすい経理業務

**仕訳入力（最優先）**

銀行明細やクレジットカード明細から、適切な勘定科目を判定して仕訳を作成する作業です。取引の8割以上は定型的（毎月同じ取引先、同じ金額、同じ勘定科目）なので、AIの得意分野です。

**請求書の作成**

定期的なクライアントへの請求書は、金額と日付をテンプレートに当てはめるだけです。請求書のPDF生成まで自動化できます。

**経費の集計・分類**

交通費、消耗品費、通信費などの経費をカテゴリ別に集計する作業です。レシートの写真からOCR（文字認識）で内容を読み取り、適切なカテゴリに分類できます。

**入金確認**

売掛金の入金状況を確認し、入金済み・未入金のリストを更新する作業です。銀行明細と請求書データを突合（照合）することで自動化できます。

**税務カレンダーの管理**

消費税の中間申告、法人税の予定納税、源泉所得税の納付期限など、税務関連の期限を管理する作業です。カレンダーに登録して、期限前にリマインドを自動送信できます。

### 自動化が難しい経理業務

**決算整理仕訳**

減価償却の計算、引当金の設定、税効果会計など、専門的な判断が必要な仕訳は、AIだけでは正確に行えません。税理士との協議が必要です。

**税務判断**

「この支出は経費になるか」「この取引の消費税区分は何か」など、税法の解釈が必要な判断は、最終的に税理士が行うべきです。

**税務調査対応**

税務署からの問い合わせや調査への対応は、人間が行う必要があります。

### 段階的に進める

レポート自動化と同じく、経理の自動化も段階的に進めます。

**フェーズ1（1ヶ月目）:** 仕訳の自動提案のみ。AIが「この取引はこの勘定科目では？」と提案し、人間が1件ずつ確認する

**フェーズ2（2〜3ヶ月目）:** 定型取引（毎月同じ取引先・金額の取引）は自動確定。非定型取引のみ人間がチェック

**フェーズ3（4ヶ月目〜）:** ほぼ全自動化。人間は月末に全体を一括レビューするだけ

私の会社は現在フェーズ3です。ここに至るまで4ヶ月かかりました。焦らず段階的に進めてください。


## AI自動仕訳の精度を92%→99%に上げた方法

自動仕訳を導入した当初、精度は92%でした。100件の仕訳のうち8件が間違っている計算です。

「92%なら悪くないのでは？」と思うかもしれません。でも、経理では1件のミスが税務申告に影響する可能性があります。92%は「使えないレベル」です。

### 精度が低かった原因

原因を分析すると、3つのパターンに集約されました。

**パターン1: 勘定科目の曖昧さ**

「Amazon.co.jp」からの購入は、「消耗品費」なのか「新聞図書費」なのか「通信費」なのか。購入した商品によって変わりますが、銀行明細には「Amazon.co.jp ¥3,980」としか表示されません。

AIはデフォルトで「消耗品費」と判定していましたが、実際には書籍の購入（新聞図書費）やAWSの利用料（通信費）も含まれていました。

**パターン2: 新規取引先の判定ミス**

初めて取引する会社の仕訳は、AIにとって「未知のパターン」です。過去のデータがないため、適切な勘定科目を推測するしかありません。

**パターン3: 例外的な取引**

通常は月額¥10,000のサービス料が、年払いの割引で¥100,000の一括支払いになったケース。AIは「¥100,000は通常と異なる」と認識しますが、正しい処理（前払費用として計上し、月割りで費用化する）まではたどり着けませんでした。

### 精度を上げた3つの改善策

**改善策1: 取引先別のルールを追加**

「Amazon.co.jp」の購入は、金額帯で仕訳を変えるルールを作りました。

- ¥5,000以下 → 消耗品費（書籍・事務用品が多い）
- ¥5,000〜¥50,000 → 確認フラグ（人間が判断）
- ¥50,000以上 → 確認フラグ（高額なので必ず人間が確認）

このように、取引先ごとに細かいルールを追加していきました。泥臭い作業ですが、効果は絶大でした。

**改善策2: 仕訳の学習データを蓄積**

AIに「過去12ヶ月分の正しい仕訳データ」を読み込ませました。Claude Codeの1Mトークンのコンテキストウィンドウを活かして、過去の全仕訳パターンを一度に分析できます。

過去のデータから「この取引先はいつもこの勘定科目」「この金額帯はこのカテゴリ」というパターンを学習し、新しい取引にも正しく適用できるようになりました。

**改善策3: 「確信度」スコアの導入**

AIに、各仕訳の「確信度」を0〜100%で出力させるようにしました。

- 確信度90%以上 → 自動確定（人間のチェック不要）
- 確信度70〜89% → 確認推奨（人間がサッと確認）
- 確信度70%未満 → 要確認（人間が必ず判断）

この仕組みにより、全体の仕訳のうち約70%が自動確定、約20%が確認推奨、約10%が要確認という配分になりました。

人間がチェックするのは全体の30%だけ。しかも、確信度が低い仕訳だけをチェックすればいいので、見落としも減りました。

これらの改善を3ヶ月かけて行った結果、精度は92%から99%に向上しました。残りの1%は、本当に例外的な取引（年に数回しか発生しないパターン）で、これは人間が処理しています。


## Claude Code Remote Tasks: クラウドで月末処理を自動実行

Claude Codeの最新機能に「Remote Tasks」があります。これは、クラウド上でClaude Codeのタスクをスケジュール実行できる機能です。

### なぜRemote Tasksが経理に向いているのか

月末の経理処理は、パソコンを起動していなくても実行したい作業です。

例えば、月末最終日の深夜0時に以下のタスクを自動実行します。

1. 銀行明細の取り込みと仕訳の自動生成
2. 当月の全仕訳の整合性チェック
3. 月次推移表の生成
4. 未入金の売掛金リストの更新
5. 翌月の税務カレンダーの確認とリマインド設定

これが終わった状態で翌朝を迎えるので、月初にはレビュー作業からスタートできます。

### Remote Tasksの設定方法

Claude Codeのダッシュボードから、以下の設定を行います。

1. 実行するタスクの内容を自然言語で記述する（「月末の経理処理一式を実行してください」）
2. 実行スケジュールを設定する（「毎月末日の23:00に実行」）
3. 実行結果の通知先を設定する（Slack、メール等）

技術的な設定は不要です。自然な日本語でタスクを記述するだけで、Claude Codeが必要な処理を自動で組み立ててくれます。


## 税理士との連携 — AI×税理士のハイブリッド運用

「AIで経理を自動化したら、税理士は不要になるのか？」

答えは明確に「No」です。

AIは「定型処理」が得意で、税理士は「判断」が得意です。この2つは補完関係にあり、どちらかが不要になるということはありません。

### AI×税理士の最適な役割分担

**AIが担当する作業:**
- 日常の仕訳入力（定型取引の自動処理）
- データの集計・分類
- 入金確認と消込
- 月次推移表の作成
- 税務カレンダーのリマインド

**税理士が担当する作業:**
- 決算整理仕訳のレビュー
- 税務判断（経費の適否、消費税区分の判断等）
- 節税アドバイス
- 税務申告書の作成・提出
- 税務調査対応

### 税理士への引き継ぎが楽になる

AIによる経理自動化の副次的な効果として、税理士への引き継ぎが格段に楽になります。

以前は、月末に仕訳データを整理して、税理士に送る作業だけで2時間かかっていました。「この取引は何ですか？」と税理士から質問が来て、それに答えるのにさらに時間がかかります。

AI自動化後は、各仕訳に「取引の説明」「分類の根拠」が自動で記録されています。税理士から質問が来ることがほとんどなくなりました。

さらに、月次の状況サマリーも自動生成されるので、税理士が状況を把握しやすくなりました。

### 税理士に自動化を説明するコツ

税理士によっては「AIが仕訳を作るなんて大丈夫なの？」と不安に思う方もいます。

私のおすすめは、最初の3ヶ月は「AIの仕訳」と「人間の仕訳」を並行して行い、差異を税理士に見せることです。

「100件の仕訳のうち、AIと人間で判断が異なったのは2件だけ。そのうち1件はAIの方が正しかった」

このような実績データを見せると、税理士も安心します。


## セキュリティ: 金融データをAIに渡す際の注意点

経理データは、問い合わせデータ以上に慎重な取り扱いが必要です。

### 基本原則

**原則1: 銀行の認証情報（ID・パスワード）はAIに渡さない**

AIに銀行の明細データを取得させる場合、API連携を使います。APIは「読み取り専用」の権限で設定し、振り込みなどの操作はAIにはできないようにします。

freeeやMoneyForwardなどの会計ソフトは、銀行口座との自動連携機能を持っています。AIは会計ソフト経由でデータを取得するので、銀行に直接アクセスする必要はありません。

**原則2: 金額データはローカル処理を優先する**

可能であれば、経理データの処理はローカル環境（自分のパソコン）で行います。クラウドに送信する場合は、SSL暗号化が施されていることを確認します。

Claude Codeはローカル実行をサポートしているため、経理データを外部に送信せずに処理できます。これは大きなメリットです。

**原則3: アクセスログを記録する**

誰が、いつ、どのデータにアクセスしたかを記録します。Claude Codeでは、実行ログが自動的に保存されるため、後から確認できます。

### 具体的なセキュリティ設定

私の会社で実施しているセキュリティ対策を公開します。

1. 経理データの保存先は暗号化されたディスクを使用
2. 会計ソフト（freee）のAPIキーは環境変数で管理（コードに直接書かない）
3. APIキーの権限は「読み取り専用」に限定
4. 月に1回、アクセスログをレビュー
5. バックアップは3世代分を保持


## コスト比較: 経理代行 vs AI自動化

経理業務を外部に委託する場合と、AI自動化する場合のコストを比較します。

### 経理代行サービスの場合

一般的な経理代行サービスの料金は、小規模法人（月50〜100仕訳）で月額¥30,000〜¥50,000程度です。記帳代行のみの場合はもう少し安く、月額¥15,000〜¥30,000程度です。

年間コスト: ¥180,000〜¥600,000

メリットは「人間が処理するので安心」「自分は何もしなくてよい」。デメリットは「コストが継続的にかかる」「タイムラグがある（月末に資料を渡して、翌月に結果が返ってくる）」。

### AI自動化の場合

初期設定: 2〜3日の作業時間（Claude Codeで設定）
月間コスト: ¥0〜¥5,000（既にClaude Code Maxを使っている場合は追加コストなし）

年間コスト: ¥0〜¥60,000

メリットは「コストが圧倒的に低い」「リアルタイムで処理される」「データが蓄積されて精度が上がる」。デメリットは「初期設定に時間がかかる」「精度のチューニングが必要」「最終確認は自分でやる必要がある」。

### どちらを選ぶべきか

私のおすすめは「AI自動化 + 税理士」のハイブリッドです。

日常の仕訳はAIが処理し、月次・決算は税理士がレビューする。この組み合わせが、コストと品質のバランスが最も良いです。

経理代行サービスは「AIの導入が本当にできない場合」の選択肢です。ITに全く抵抗がある場合や、社長が経理に全く時間を使いたくない場合は、経理代行の方が良いケースもあります。


## 実装のステップバイステップ

最後に、経理自動化の具体的な導入手順をまとめます。

### 1週目: 現状把握

- 過去3ヶ月分の仕訳データを確認する
- 取引先と勘定科目のパターンを洗い出す
- 定型取引（毎月同じ内容）と非定型取引の割合を把握する

### 2週目: AI仕訳の設定

- Claude Codeに過去の仕訳データを読み込ませる
- 取引先別の仕訳ルールを設定する
- テスト用の取引データで精度を確認する

### 3〜4週目: 並行運用

- AIの仕訳と手動の仕訳を同時に行う
- 差異を記録し、AIの改善ポイントを特定する
- 税理士にAI仕訳の結果を見せて、フィードバックをもらう

### 2ヶ月目: 段階的移行

- 定型取引のみAI自動化に切り替え
- 非定型取引は引き続き手動
- 月末に全体をレビュー

### 3ヶ月目以降: 本格運用

- 確信度スコアの導入
- 確信度90%以上の仕訳は自動確定
- 月末のレビュー時間を30分以内に短縮

この流れで進めれば、3ヶ月後には「月末経理が30分で終わる」状態になっているはずです。


## この章のまとめ

経理の自動化は、3つの最優先業務の中で最もインパクトが大きい。

- 月末5.5時間の作業が30分に短縮（91%削減）
- AI仕訳の精度は、チューニングにより99%まで向上可能
- 税理士との役割分担を明確にし、ハイブリッド運用が最適
- セキュリティは「読み取り専用API」「ローカル処理」「アクセスログ」の3原則

次章では、これら3つの業務を自動化するためのツール選定を詳しく解説します。月¥32,280で98%を自動化した実際のツールスタックを公開します。


## この章のアクションリスト

1. 現在の月末経理にかかっている時間を計測してください。「何時間かかっているか」を正確に把握することが第一歩です
2. 過去3ヶ月の仕訳データを確認し、定型取引（毎月同じパターン）の割合を調べてください。8割以上が定型なら、自動化の効果が大きいです
3. 会計ソフト（freee、MoneyForward等）のAPI連携機能を確認してください。API連携があれば、自動化の実装が大幅に楽になります
4. 税理士に「AI自動仕訳の導入を検討している」と相談してください。税理士の理解と協力が、成功の鍵です
5. 最初の1ヶ月は必ず「手動と並行運用」してください。AIの精度を信頼できるレベルまで確認してから、切り替えます
