# 6ヶ月DXロードマップ — 月5万円から始める段階的自動化

## 最後に、全体を1枚の地図にまとめます

ここまで11章にわたって、中小企業のDX自動化について解説してきました。技術的な方法、運用の設計、ROI計算、セキュリティ、社内の巻き込み方。必要な知識はすべてお伝えしました。

しかし、「全部読んだけど、結局どこから手をつければいいの？」と思っている方もいるかもしれません。

この最終章では、**月5万円以内の予算で、6ヶ月かけて段階的にDXを進めるロードマップ**をお渡しします。このロードマップの通りに進めれば、6ヶ月後には月20-40時間の業務削減が実現しているはずです。

## Month 1: 業務棚卸し + ツール選定（投資 ¥0）

### 目的

最初の1ヶ月は、**お金を一切使わずに「準備」に集中**します。ここを飛ばして「とりあえずツールを導入しよう」とすると、第8章のD社のように「使われないツール」に月額費用を払い続けることになります。

### やること

**第1週: 業務の棚卸し**

社内の全業務をリストアップし、以下の4項目で評価します。

| 業務名 | 月間所要時間 | 担当者 | 自動化の可能性 |
|--------|------------|--------|--------------|
| 日次売上レポート作成 | 11時間 | 田中 | 高 |
| 請求書発行 | 5時間 | 佐藤 | 高 |
| 問い合わせ対応 | 40時間 | 鈴木 | 中 |
| 在庫確認 | 8時間 | 田中 | 高 |
| 商談・営業 | 60時間 | 社長 | 低 |
| 企画・戦略立案 | 20時間 | 社長 | 低 |

「自動化の可能性」の判断基準は以下の通りです。

- **高**: 定型的で、毎回同じ手順で行う業務（レポート作成、データ入力、定型メール）
- **中**: ある程度のパターンがあるが、判断が必要な業務（問い合わせ対応、仕入れ判断）
- **低**: 創造性や高度な判断が必要な業務（営業、企画、戦略）

**第2週: 自動化の優先順位を決める**

棚卸しの結果から、第3章で解説した「最初にやるべき3つ」を選定します。

優先順位の判断基準はこの3つです。

1. **月間所要時間が大きい**（削減効果が高い）
2. **自動化の可能性が「高」**（技術的に実現しやすい）
3. **失敗してもリスクが低い**（社内業務であり、社外に影響しない）

3つの条件をすべて満たす業務が、最初の自動化対象です。多くの場合、「日次レポート作成」がここに該当します。

**第3週: ツールの調査**

自動化に使えるツールを調査します。この段階では契約しません。無料トライアルで試すだけです。

中小企業に適したツールの目安は以下の通りです。

| ツール | 用途 | 月額目安 |
|--------|------|---------|
| Google スプレッドシート + GAS | レポート自動化 | ¥0 |
| Claude Code | 汎用AI自動化 | ¥3,000-30,000 |
| Zapier（Free） | ツール間連携 | ¥0 |
| Notion | タスク・情報管理 | ¥0-1,650 |
| チャットボットサービス | 問い合わせ対応 | ¥3,000-10,000 |

**第4週: チャンピオンユーザーの選定**

第11章で解説した方法で、チャンピオンユーザー候補を1-2名特定します。

### Month 1 チェックリスト

- 全業務の棚卸しが完了したか
- 最初に自動化する業務を1つ決めたか
- 使用するツールの候補を2-3個ピックアップしたか
- チャンピオンユーザー候補を特定したか
- AI利用ガイドライン（第10章のテンプレート）を作成したか

### 投資額: ¥0

## Month 2: 日次レポートの自動化（投資 ¥0-5,000）

### 目的

最初の自動化を実行し、**「成功体験」を作る**月です。ここでの成功が、その後のDX推進の原動力になります。

### やること

**第1週: 現状の計測**

自動化する前に、必ず「現在の状態」を数字で記録します。

- 現在のレポート作成時間: ○分/日
- 月間の合計時間: ○時間
- ミスの頻度: 月○回
- レポートの配信時刻: 毎日○時ごろ

この数字が、導入後の効果測定のベースラインになります。

**第2週: 自動化の実装**

Google スプレッドシートとGoogle Apps Script（GAS）を使う場合は無料です。Claude Codeを使う場合は月¥3,000-5,000程度です。

第4章で解説した方法に沿って、日次レポートの自動生成を実装します。チャンピオンユーザーと一緒に進めてください。

**第3週: テスト運用**

自動生成されたレポートと、手動で作成したレポートを1週間並行して比較します。差異がないことを確認してから、自動版に切り替えます。

**第4週: 本番運用開始 + 効果計測**

自動化を本番運用に切り替えます。同時に、以下の数字を記録します。

- 自動化後のレポート確認時間: ○分/日
- 月間の合計時間: ○時間（削減量を計算）
- ミスの頻度: 月○回
- レポートの配信時刻: 毎日○時（正確に定時配信されているか）

### Month 2 チェックリスト

- 自動化前のベースライン数字を記録したか
- 自動化の実装が完了したか
- 1週間のテスト運用で問題がないことを確認したか
- 本番運用を開始したか
- Before/Afterの数字を全社に共有したか

### 投資額: ¥0-5,000

## Month 3: 問い合わせ対応の半自動化（投資 ¥5,000）

### 目的

2つ目の自動化に着手します。問い合わせ対応は、**自動化の効果が最もわかりやすい業務の1つ**です。

### やること

**第1週: 問い合わせの分類**

過去1ヶ月分の問い合わせをすべてリストアップし、カテゴリに分類します（第5章参照）。

目標は、「よくある質問」（自動回答可能）と「個別対応」（人間が必要）の比率を把握することです。筆者の経験では、60-70%が自動回答可能なケースが多いです。

**第2週: FAQ + 自動応答の準備**

よくある質問の回答集を作成します。これをWebフォームやチャットボットに組み込みます。

完璧なFAQを作る必要はありません。まず上位10個の質問と回答を用意するだけで、問い合わせの40-50%をカバーできます。

**第3週: 導入とテスト**

Webフォーム + 自動応答を設置し、1週間テストします。自動応答の精度を確認し、不足している回答を追加します。

**第4週: 本番運用 + 効果計測**

電話やメールでの問い合わせを、Webフォーム経由に誘導します。「お問い合わせはこちらのフォームから」とWebサイトや名刺に記載。

電話対応を完全になくす必要はありません。Webフォームで対応できない案件は、これまで通り電話で対応します。

### Month 3 チェックリスト

- 問い合わせの分類が完了したか
- 上位10個のFAQを作成したか
- Webフォーム + 自動応答を設置したか
- テスト運用で精度を確認したか
- 問い合わせの何%が自動応答で完結しているか計測しているか

### 投資額: ¥5,000（チャットボットサービスの月額）

## Month 4: 経理処理の自動化開始（投資 ¥5,000-10,000）

### 目的

3つ目の自動化として、経理処理に着手します。第6章で解説した方法に沿って進めます。

### やること

**第1週: 現在の経理フローを可視化**

誰が、何を、どのツールで処理しているかを書き出します。

多くの中小企業では、以下のようなフローになっているはずです。

```
領収書を受け取る → Excelに入力 → 勘定科目を判断 → 
会計ソフトに入力 → 月末に税理士に送付
```

この中で、「Excelに入力」と「勘定科目の判断」が自動化の対象です。

**第2週: 自動仕訳のルール設定**

よく使う取引先と勘定科目の対応表を作成します。

| 取引先 | 勘定科目 |
|--------|---------|
| Amazon | 消耗品費 or 仕入 |
| 東京電力 | 水道光熱費 |
| NTT | 通信費 |
| ○○不動産 | 地代家賃 |

この対応表をAIに学習させれば、「Amazonからの¥3,000の請求 → 消耗品費」を自動で判定できます。

**第3週: テスト運用**

1ヶ月分の過去データでテストします。AIが自動分類した結果と、人間が分類した結果を比較。精度が95%以上になるまで調整します。

**第4週: 本番運用開始**

新しい取引から自動分類を適用します。ただし、**最初の1ヶ月は人間が全件チェック**してください（第7章のdraftモード）。問題がなければ、翌月から例外のみチェックに切り替えます。

### Month 4 チェックリスト

- 現在の経理フローを可視化したか
- 取引先と勘定科目の対応表を作成したか
- 過去データでテストし、精度95%以上を確認したか
- 本番運用を開始したか（人間チェック付き）
- 税理士に「AIを使った自動仕訳を始めます」と連絡したか

### 投資額: ¥5,000-10,000（AIツール + 会計ソフト連携）

## Month 5: 全体の効果測定 + 改善（投資変わらず）

### 目的

新しい自動化は追加せず、**ここまでの3つの自動化の効果を正確に測定し、改善する**月です。

「走りながら振り返る」のがDXの鉄則です。4ヶ月間走り続けたので、一度立ち止まります。

### やること

**第1週: 効果測定**

Month 2-4で導入した3つの自動化について、以下の数字をまとめます。

| 自動化対象 | Before | After | 削減時間 | 削減率 |
|-----------|--------|-------|---------|--------|
| 日次レポート | ○時間/月 | ○時間/月 | ○時間 | ○% |
| 問い合わせ対応 | ○時間/月 | ○時間/月 | ○時間 | ○% |
| 経理処理 | ○時間/月 | ○時間/月 | ○時間 | ○% |
| **合計** | | | **○時間** | |

同時に、投資コストも集計します。

| 項目 | 月額 |
|------|------|
| AIツール | ¥○ |
| チャットボット | ¥○ |
| その他 | ¥○ |
| **合計** | **¥○** |

第9章の計算式でROIを算出してください。

**第2週: 社内フィードバック収集**

全社員に以下の質問をします（5分のアンケートで十分です）。

1. 自動化された業務で、便利になったと感じるものは？
2. 使いにくいと感じる点は？
3. 他に自動化してほしい業務は？
4. AIの利用で不安に感じることは？

**第3週: 改善の実施**

フィードバックに基づいて改善します。よくある改善点は以下の通りです。

- レポートの項目を追加・変更してほしい → テンプレート修正
- 自動応答の回答が的外れなケースがある → FAQの追加・修正
- 経理の自動分類で間違いが多い取引がある → ルールの追加

**第4週: 効果報告会**

全社ミーティングで、5ヶ月間の成果を共有します。数字で示すことが重要です。

「5ヶ月で月○時間の業務を削減しました。月額コストは¥○です。年間換算で¥○のコスト削減になります」

この報告が、次のMonth 6の投資を承認してもらう布石にもなります。

### Month 5 チェックリスト

- 3つの自動化のBefore/After数字をまとめたか
- ROIを計算したか
- 全社員からフィードバックを収集したか
- フィードバックに基づく改善を実施したか
- 効果報告会を開催したか

### 投資額: 変わらず（Month 4と同じ）

## Month 6: 次の自動化対象を選定 + 社内展開（投資 ¥30,000-50,000）

### 目的

最初の3つの自動化が定着したことを確認し、**次の展開を計画する**月です。ここで投資を増やすかどうかは、Month 5の効果測定の結果次第です。

### やること

**第1週: 次の自動化候補を3つ選ぶ**

Month 1の業務棚卸しに戻り、次に自動化すべき業務を3つ選定します。Month 5のアンケートで「自動化してほしい」という声が多かった業務を優先します。

よく候補に挙がるのは以下の業務です。

- 在庫管理の自動化
- 顧客管理（CRM）の導入
- 社内の情報共有（ナレッジベース）の整備
- マーケティング（SNS投稿、メールマガジン）の半自動化
- 勤怠管理の自動化

**第2週: 投資判断**

次の自動化に必要な投資を見積もり、ROIを計算します（第9章の方法で）。

Month 5の効果が明確に出ていれば、経営者は追加投資に前向きになるはずです。「最初の¥10,000/月の投資で月○時間を削減できた。次は¥30,000-50,000/月の投資で、さらに○時間削減できる見込みです」と提案します。

**第3週: 他部署への横展開**

ここまで1つの部署（またはチーム）で進めてきた自動化を、他の部署にも展開します。チャンピオンユーザーが教える形で進めると、スムーズです。

**第4週: 7ヶ月目以降の計画策定**

半年間の実績をもとに、今後6ヶ月の計画を立てます。

ただし、計画は「大まかな方針」で十分です。具体的な内容は、その時々の効果測定とフィードバックに基づいて調整します。

### Month 6 チェックリスト

- 次に自動化する業務候補を3つ選定したか
- 各候補のROIを計算したか
- 他部署への横展開を開始したか
- 7ヶ月目以降の大まかな計画を策定したか
- 半年間の成果を経営会議（または全社ミーティング）で報告したか

### 投資額: ¥30,000-50,000（新しいツールの追加契約）

## 6ヶ月後に期待できる成果

このロードマップに沿って進めた場合、6ヶ月後の期待成果は以下の通りです。

### 業務時間の削減

| 自動化対象 | 削減時間（月間）|
|-----------|----------------|
| 日次レポート | 10-15時間 |
| 問い合わせ対応 | 15-25時間 |
| 経理処理 | 5-10時間 |
| **合計** | **30-50時間** |

月30-50時間の削減は、**営業日換算で約4-6日分**に相当します。毎月4-6日分の業務時間が浮くのです。

### コスト削減

| 項目 | 金額（年間）|
|------|-----------|
| 人件費削減（30時間 × ¥2,500 × 12ヶ月） | ¥900,000 |
| 外注費削減（経理代行の減額等） | ¥200,000-400,000 |
| ミス削減による損失回避 | ¥50,000-100,000 |
| **合計** | **¥1,150,000-1,400,000** |

6ヶ月間の投資総額は約¥150,000-300,000です。年間のコスト削減が¥1,150,000以上であれば、**ROIは380%以上**です。

### 定性的な成果

数字に表れない成果も重要です。

- **属人化の解消**: 特定の社員に依存しない業務フローが確立される
- **ミスの減少**: 手作業に起因するミスが大幅に減る
- **社員の満足度向上**: 単純作業から解放され、より創造的な仕事に集中できる
- **意思決定の高速化**: リアルタイムのデータに基づいた経営判断が可能になる
- **DXの文化醸成**: 「自動化は怖い」から「自動化は便利」への意識変革が定着する

## 著者からのメッセージ: 完璧を目指さない。80点で始めて、走りながら改善する

最後に、筆者からのメッセージをお伝えします。

DXは「完璧な計画を立ててから始める」ものではありません。**80点の状態で始めて、走りながら改善していく**ものです。

筆者自身、最初にAIを業務に導入したとき、うまくいかないことだらけでした。レポートのフォーマットがずれる、自動応答が的外れな回答をする、仕訳の分類を間違える。

しかし、1つひとつ修正し、2週間後にはほぼ問題なく動くようになりました。3ヶ月後には「これがない状態にはもう戻れない」と感じるようになりました。

今では月¥32,280の投資で、11部門のAIエージェントが24時間稼働し、自動化率98%を達成しています。しかし、これは最初から目指した姿ではありません。小さな自動化を1つずつ積み重ねた結果です。

あなたの会社のDXも、**最初の一歩は「日次レポートの自動化」で十分**です。それが成功したら次に進む。1つずつ、確実に。

この本が、その最初の一歩を踏み出すきっかけになれば、著者としてこれ以上の喜びはありません。
