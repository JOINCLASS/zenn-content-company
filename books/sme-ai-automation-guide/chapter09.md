# ノーコード/ローコードで作る業務アプリ -- Bubble/Retool活用

## はじめに -- 「ちょうどいいツールがない」問題を解決する

第4〜8章では、既存のSaaSやAIツールを活用した業務自動化を解説した。しかし、中小企業の業務には、既存ツールでは対応しきれない「自社独自のフロー」が必ず存在する。

よくある例:
- 「この承認フロー、どのワークフローツールにも合わない」
- 「Excelで管理している顧客情報と案件情報を統合したいが、市販のCRMではカラムが足りない」
- 「現場のスタッフが使う点検チェックリストをアプリ化したいが、開発予算がない」

こうした「かゆいところに手が届かない」問題を解決するのが、ノーコード/ローコードツールだ。プログラミングの知識なしに（または最小限のコードで）、自社専用の業務アプリを作れる。

外部にWebアプリの開発を委託すると、最低でも100〜500万円。ノーコードツールを使えば、月額1〜3万円のツール代と、自分の作業時間だけで済む。


## ノーコードとローコードの違い

| 項目 | ノーコード | ローコード |
|------|----------|----------|
| コードの必要性 | 完全に不要 | 一部のカスタマイズにコードが必要 |
| 柔軟性 | 中（ツールの機能範囲内） | 高（コードで拡張可能） |
| 学習コスト | 低（数日〜1週間） | 中（1〜2週間） |
| 向いている用途 | シンプルな業務アプリ | 複雑なロジックのある業務アプリ |
| 代表的なツール | Bubble, AppSheet, Glide | Retool, Tooljet |


## ツール選定ガイド

本章では、**Bubble**（ノーコード）と**Retool**（ローコード）の2つに絞って解説する。この2つを選んだ理由は以下の通り。

| 基準 | Bubble | Retool |
|------|--------|--------|
| 得意な用途 | 顧客向けWebアプリ、社外公開アプリ | 社内管理ツール、ダッシュボード |
| UIのカスタマイズ性 | 非常に高い | 高い（テンプレートベース） |
| データベース | 内蔵DB | 外部DB接続（PostgreSQL等） |
| 無料プラン | あり（開発・テストは無料） | あり（5ユーザーまで無料） |
| 日本語の学習リソース | 豊富（YouTube、Udemy等） | やや少ない |

**選び方の目安:**

- **社外の人が使うアプリを作りたい** → Bubble
- **社内のスタッフが使う管理ツールを作りたい** → Retool
- **スプレッドシートの延長で作りたい** → AppSheet（Google製）
- **スマホアプリを作りたい** → Glide


## Bubble入門 -- 顧客向けWebアプリを作る

### Bubbleとは

Bubbleは、2012年にニューヨークで創業されたノーコードプラットフォームだ。ドラッグ&ドロップの操作だけで、本格的なWebアプリケーションを構築できる。

データベースの設計、ユーザー認証、API連携、レスポンシブデザインなど、通常ならプログラマーが必要な機能を、すべてビジュアルエディタで実現できる。

### Bubbleの料金プラン

| プラン | 月額費用 | 主な制限 |
|--------|---------|---------|
| Free | 無料 | Bubble ブランドロゴ表示。API連携に制限 |
| Starter | $29 | カスタムドメイン。API連携可能 |
| Growth | $119 | 本番運用向け。パフォーマンス向上 |
| Team | $349 | チーム開発向け。バージョン管理 |

**中小企業の業務アプリなら、Starterプラン（月約4,400円）で十分。** 開発・テスト段階は無料プランで行い、本番公開時にStarterに切り替える。

### 実践: 顧客管理アプリを作る

HubSpotやSalesforceでは物足りない、自社独自の顧客管理アプリをBubbleで作る例を紹介する。

**要件:**

- 顧客情報（会社名、担当者名、連絡先、業種、取引ステータス）を管理
- 商談履歴を時系列で記録
- 見積書のPDFを添付して保管
- 営業担当者ごとの案件一覧を表示
- スマホからも閲覧・入力可能

**手順1: プロジェクトの作成**
1. bubble.io にアクセスし、無料アカウントを作成
2. 「Create a new app」をクリック
3. アプリ名を入力（例: customer-management）
4. テンプレートは「Start from scratch」を選択

**手順2: データベースの設計**

Bubbleのデータタブで、以下のデータ型（Type）を作成する。

```
Type: Customer（顧客）
  Fields:
    - company_name (text): 会社名
    - contact_name (text): 担当者名
    - email (text): メールアドレス
    - phone (text): 電話番号
    - industry (text): 業種
    - status (text): ステータス（見込み/商談中/成約/失注）
    - assigned_to (User): 担当営業
    - created_date (date): 登録日

Type: Deal（商談）
  Fields:
    - customer (Customer): 関連顧客
    - title (text): 商談名
    - amount (number): 金額
    - stage (text): ステージ
    - note (text): メモ
    - attachments (file list): 添付ファイル
    - deal_date (date): 商談日
```

**手順3: 画面（ページ）の設計**

Bubbleのデザインタブで、以下のページを作成する。

**ページ1: 顧客一覧ページ**
1. Repeating Group（繰り返しグループ）コンポーネントを配置
2. データソースを「Do a search for → Customer」に設定
3. 各セルに会社名、担当者名、ステータスを表示するテキスト要素を配置
4. 検索バーを追加（会社名で絞り込み）
5. 「新規登録」ボタンを配置

**ページ2: 顧客詳細ページ**
1. URLパラメータから顧客IDを受け取る
2. 顧客情報の表示エリア（会社名、連絡先等）
3. 編集ボタン → ポップアップで編集フォーム表示
4. 商談履歴のRepeating Group（時系列表示）
5. 新規商談の登録フォーム

**ページ3: ダッシュボード**
1. ステータス別の顧客数（パイチャート）
2. 月別の商談金額（棒グラフ）
3. 担当者別の案件数

**手順4: ワークフロー（ロジック）の設定**

Bubbleのワークフロータブで、ボタンクリック時の動作を設定する。

例: 「新規顧客登録」ボタンのワークフロー
```
When: Button "新規登録" is clicked
Step 1: Create a new thing → Customer
  - company_name = Input Company Name's value
  - contact_name = Input Contact Name's value
  - email = Input Email's value
  - phone = Input Phone's value
  - industry = Dropdown Industry's value
  - status = "見込み"
  - assigned_to = Current User
  - created_date = Current date/time
Step 2: Navigate to → 顧客詳細ページ（新しく作成したCustomerのIDを渡す）
Step 3: Show → Alert "登録が完了しました"
```

**手順5: レスポンシブ対応**

Bubbleのレスポンシブエディタで、スマホ表示時のレイアウトを調整する。テーブル表示をカード表示に変更したり、不要な列を非表示にしたりする。

### Bubbleで作る業務アプリの他の例

| 用途 | 概要 | 想定制作時間 |
|------|------|------------|
| 予約管理システム | カレンダー表示+顧客管理+自動リマインド | 2〜3週間 |
| 社内稟議・承認ワークフロー | 申請→承認→差戻しのフロー管理 | 1〜2週間 |
| 在庫管理ダッシュボード | バーコードスキャン+在庫数リアルタイム表示 | 2〜3週間 |
| イベント参加者管理 | 申込受付→決済→QRコード発行→受付チェックイン | 3〜4週間 |


## Retool入門 -- 社内管理ツールを爆速で作る

### Retoolとは

Retoolは、社内ツールの構築に特化したローコードプラットフォームだ。Bubbleが「何でも作れる汎用プラットフォーム」であるのに対し、Retoolは「社内の管理画面を最速で作る」ことに最適化されている。

**Retoolの強み:**

- テーブル、フォーム、チャートなどの部品があらかじめ用意されている
- PostgreSQL、MySQL、MongoDB、Google Sheets、REST APIなど、あらゆるデータソースに接続可能
- SQLやJavaScriptで細かいカスタマイズが可能（ローコード）
- 管理画面の構築が非常に速い（数時間で実用的なツールが作れる）

### Retoolの料金プラン

| プラン | 月額費用 | 主な制限 |
|--------|---------|---------|
| Free | 無料 | 5ユーザーまで。月500クエリ |
| Team | $10/ユーザー | 無制限クエリ。カスタムブランディング |
| Business | $50/ユーザー | 監査ログ。SSO対応 |

**中小企業の社内ツールなら、Freeプラン（5ユーザーまで無料）で十分始められる。**

### 実践: 受注管理ダッシュボードを作る

Excelで管理している受注データを、Retoolでダッシュボード化する例を紹介する。

**要件:**

- 受注一覧の表示（フィルタ、ソート、検索機能つき）
- 受注ステータスの変更（未確認→確認済み→出荷済み→完了）
- 月別の受注金額のグラフ表示
- ステータス別の件数サマリ

**手順1: データソースの接続**

Google Sheetsを使っている場合:
1. Retoolにログイン
2. 「Resources」→「New Resource」をクリック
3. 「Google Sheets」を選択
4. Googleアカウントを連携

PostgreSQLなどのデータベースを使っている場合:
1. 「Resources」→「New Resource」をクリック
2. 「PostgreSQL」を選択
3. ホスト名、ポート、データベース名、認証情報を入力

**手順2: テーブルコンポーネントの配置**

1. 「Create new app」をクリック
2. 左パネルから「Table」コンポーネントをキャンバスにドラッグ
3. テーブルのデータソースを設定

Google Sheetsの場合:
```
Resource: Google Sheets
Spreadsheet: 受注管理
Sheet: Sheet1
```

SQLデータベースの場合:
```sql
SELECT
  order_id,
  customer_name,
  product_name,
  quantity,
  amount,
  status,
  order_date
FROM orders
ORDER BY order_date DESC
```

4. テーブルが自動的にデータを読み込んで表示される

**手順3: フィルタと検索の追加**

1. 「Select」コンポーネントを追加（ステータスフィルタ用）
   - Options: 全件, 未確認, 確認済み, 出荷済み, 完了
2. 「Text Input」コンポーネントを追加（検索用）
3. テーブルのクエリにフィルタ条件を追加

```sql
SELECT * FROM orders
WHERE
  ({{ !statusFilter.value || statusFilter.value === '全件' }} OR status = {{ statusFilter.value }})
  AND
  ({{ !searchInput.value }} OR customer_name ILIKE {{ '%' + searchInput.value + '%' }})
ORDER BY order_date DESC
```

**手順4: ステータス変更機能の追加**

1. テーブルの行をクリックしたときに、サイドパネルを表示
2. サイドパネルにステータス変更用の「Select」コンポーネントを配置
3. 「更新」ボタンを追加し、クリック時に以下のクエリを実行

```sql
UPDATE orders
SET status = {{ statusSelect.value }}
WHERE order_id = {{ table1.selectedRow.data.order_id }}
```

**手順5: グラフの追加**

1. 「Chart」コンポーネントをキャンバスにドラッグ
2. データソースに集計クエリを設定

```sql
SELECT
  DATE_TRUNC('month', order_date) as month,
  SUM(amount) as total_amount
FROM orders
GROUP BY month
ORDER BY month
```

3. グラフの種類を「棒グラフ」に設定
4. X軸: month、Y軸: total_amount

これで、受注データのダッシュボードが完成する。作業時間は1〜2時間程度だ。

## kintone -- 日本企業向けのもう1つの選択肢

Bubble や Retool は海外製のツールだが、日本企業には **kintone（キントーン）** という選択肢もある。

### kintoneの特徴

- サイボウズ（日本企業）が開発・運営
- 日本語のサポート、ドキュメント、コミュニティが充実
- 業務アプリをドラッグ&ドロップで作成
- 月額1,500円/ユーザー（スタンダードコース）
- 30日間の無料トライアルあり

### kintoneが向いているケース

- 社内の情報共有・申請管理が主な用途
- IT担当者がいない（またはITに詳しくない）企業
- 日本語のサポートを重視する場合
- サイボウズの他の製品（Garoon、メールワイズ）を使っている場合

### kintoneが向いていないケース

- 顧客向けの外部公開アプリを作りたい場合（kintoneは社内利用が前提）
- 高度なUI/UXカスタマイズが必要な場合
- ユーザー数が少ない場合（1人あたりの費用が割高）


## ノーコード/ローコード開発の注意点

### 注意点1: スケーラビリティの限界

ノーコードツールは、小〜中規模のアプリに最適だ。しかし、データ量が数十万件を超えたり、同時接続ユーザーが数百人を超えたりすると、パフォーマンスの問題が出ることがある。

**対策:** 最初はノーコードで素早くプロトタイプを作り、本格運用で限界を感じたら、プログラミングで作り直すという段階的なアプローチが現実的だ。

### 注意点2: ベンダーロックイン

ノーコードツールで作ったアプリは、そのツールのプラットフォーム上でしか動作しない。ツールのサービスが終了したり、大幅な値上げがあった場合、移行が困難になる。

**対策:** データは定期的にCSVやAPI経由でバックアップしておく。ビジネスロジックの仕様書を文書化しておく。

### 注意点3: セキュリティ

顧客の個人情報や機密データを扱う場合は、ツールのセキュリティ対策を確認する。

確認すべきポイント:
- データの暗号化（保存時、通信時）
- アクセス制御（ロールベースのアクセス権限）
- 監査ログ（誰がいつ何をしたか）
- バックアップ（データの復元が可能か）
- SOC2/ISO27001 などのセキュリティ認証


## まとめ

本章のポイントを整理する。

1. **「ちょうどいいツールがない」問題は、ノーコード/ローコードで解決できる**
2. **Bubbleは顧客向けWebアプリ、Retoolは社内管理ツールの構築に最適**
3. **kintoneは日本企業向けの現実的な選択肢**
4. **外部開発委託（100〜500万円）の代わりに、月額1〜3万円のツール代で自社開発が可能**
5. **まずは小さなアプリ（1機能）から始め、使いながら改善していくアプローチが成功率を高める**
6. **スケーラビリティ、ベンダーロックイン、セキュリティの3つのリスクを事前に認識しておく**
