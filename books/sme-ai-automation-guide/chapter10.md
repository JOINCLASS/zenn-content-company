# API連携とワークフロー自動化 -- Zapier/Make/n8n実践

## はじめに -- ツールとツールを「つなぐ」ことで自動化は加速する

第4〜9章で紹介した各種ツール（クラウド会計、CRM、チャットボット、ノーコードアプリ等）は、単体でも十分に効果がある。しかし、**ツール同士を連携させると、自動化の効果は何倍にもなる。**

例えば、以下のような連携だ。

- Webフォームに問い合わせが来る → CRMに顧客情報を自動登録 → 営業担当にSlack通知 → お礼メールを自動送信
- freeeで請求書を発行する → 顧客にメール送信 → スプレッドシートに記録 → 入金期日の3日前にリマインド通知
- Shopifyで注文が入る → 在庫数を自動更新 → 在庫が発注点を下回ったらSlackにアラート → 倉庫に出荷指示

これらの連携を実現するのが、ワークフロー自動化ツール（Zapier、Make、n8n）だ。

本章では、ワークフロー自動化の基本概念から、具体的なシナリオの構築手順まで、実践的に解説する。


## ワークフロー自動化の基本概念

### トリガーとアクション

ワークフロー自動化の基本構造は、**「トリガー」と「アクション」**の2つだ。

- **トリガー**: 自動化を開始するきっかけ（例: メールを受信した、フォームに入力があった）
- **アクション**: トリガーに反応して実行する処理（例: Slackに通知する、スプレッドシートに行を追加する）

```
トリガー → アクション1 → アクション2 → アクション3
  (きっかけ)    (処理1)      (処理2)      (処理3)
```

例:
```
Googleフォームに回答が来る（トリガー）
  → HubSpot CRMに顧客を登録（アクション1）
  → Slackの営業チャンネルに通知（アクション2）
  → 回答者にお礼メールを送信（アクション3）
```

### APIとは

API（Application Programming Interface）は、ソフトウェア同士が情報をやり取りするための窓口だ。

わかりやすく言えば、APIは「レストランの注文カウンター」のようなものだ。

- あなた（ワークフローツール）がカウンターで注文する（リクエストを送る）
- キッチン（SaaSのサーバー）が料理を作る（データを処理する）
- カウンターで料理が出てくる（レスポンスが返る）

ワークフロー自動化ツールは、このAPIを使ってツール間のデータを橋渡しする。**APIの仕組みを理解する必要はない**。Zapier、Make、n8nが裏側でAPIのやり取りをすべて代行してくれるからだ。


## ツール比較: Zapier vs Make vs n8n

### 詳細比較

| 項目 | Zapier | Make | n8n |
|------|--------|------|-----|
| 月額費用 | $19.99〜 | $9〜 | 無料（セルフホスト） |
| 対応アプリ数 | 7,000以上 | 1,800以上 | 500以上（+カスタム） |
| UIの使いやすさ | リスト形式。シンプル | ビジュアル。フローが見える | ビジュアル。n8nクラウドは使いやすい |
| 無料プラン | 5 Zaps、100タスク/月 | 1,000オペレーション/月 | セルフホストなら完全無料 |
| エラー処理 | 基本的 | 高度（リトライ、代替ルート） | 高度 |
| 条件分岐 | あり（有料プラン） | あり（無料プランから） | あり |
| 日本語サポート | なし | なし | なし |
| 学習リソース（日本語） | 豊富 | やや豊富 | 増加中 |

### 選び方

- **「まず簡単に始めたい」** → Zapier（UIが最もシンプル）
- **「コスパ重視でしっかり作りたい」** → Make（安価で高機能）
- **「コストを最小限にしたい」「自社サーバーがある」** → n8n（無料）
- **「Microsoft 365環境」** → Power Automate（Microsoftとの親和性が高い）


## Zapierの実践

### シナリオ1: 問い合わせフォーム → CRM登録 → Slack通知

**要件:**
Webサイトの問い合わせフォーム（Googleフォーム）に入力があったら、HubSpot CRMに登録して、Slackの営業チャンネルに通知する。

**構築手順:**

**Step 1: Zapを作成する**
1. Zapierにログイン
2. 「Create Zap」をクリック

**Step 2: トリガーを設定する**
1. App: Google Forms
2. Event: New Response in Spreadsheet
3. Googleアカウントを連携
4. 対象のフォームとスプレッドシートを選択
5. テスト: 実際にフォームに回答を入力してテスト

**Step 3: アクション1（CRM登録）を設定する**
1. App: HubSpot
2. Event: Create Contact
3. HubSpotアカウントを連携
4. フィールドマッピング:
   - Email: フォームのメールアドレス欄
   - First Name: フォームの名前欄
   - Company: フォームの会社名欄
   - Phone: フォームの電話番号欄
5. テスト: HubSpotに正しくコンタクトが作成されるか確認

**Step 4: アクション2（Slack通知）を設定する**
1. App: Slack
2. Event: Send Channel Message
3. Slackワークスペースを連携
4. Channel: #営業
5. メッセージ:
```
新しい問い合わせがありました

会社名: {{Google Forms のデータ}}
担当者名: {{Google Forms のデータ}}
メール: {{Google Forms のデータ}}
問い合わせ内容: {{Google Forms のデータ}}

HubSpot: {{HubSpot の Contact URL}}
```

**Step 5: Zapを有効化**
1. テストがすべて成功したら、「Turn on Zap」で有効化
2. 以降、フォームに回答が来るたびに自動で処理される

### シナリオ2: 請求書発行 → メール送信 → リマインド設定

**要件:**
freeeで請求書を発行したら、顧客にメールで送信し、支払期日の3日前にSlackでリマインド通知を受け取る。

**構築手順:**

1. トリガー: freee → New Invoice（新しい請求書の作成）
2. アクション1: Gmail → Send Email（顧客にメール送信）
   - To: 請求先のメールアドレス
   - Subject: 「請求書送付のご案内【{会社名}】{請求書番号}」
   - Body: 丁寧な文面 + PDFの添付リンク
3. アクション2: Google Sheets → Create Row（請求書台帳に記録）
   - 請求書番号、顧客名、金額、発行日、支払期日
4. アクション3: Zapier Schedule → Delay Until（支払期日の3日前まで待機）
5. アクション4: Slack → Send Message（リマインド通知）
   - 「{顧客名}の支払期日が3日後です。金額: {金額}円」


## Makeの実践

### Makeの特徴: ビジュアルフローエディタ

Makeの最大の強みは、ワークフローを**ビジュアルなフロー図**として設計できる点だ。Zapierが「リスト形式（上から下に流れる）」のに対し、Makeは「ノード形式（線でつながった図）」で表現される。

これにより、分岐や合流を含む複雑なワークフローも直感的に設計できる。

### シナリオ: メール分類 + 自動対応パイプライン

第5章で紹介した「メール仕分け・対応の自動化」をMakeで構築する。

**フロー図:**
```
Gmail（新着メール受信）
  │
  ▼
OpenAI（メール分類）
  │
  ├─ [見積依頼] ─→ Slack通知（営業チャンネル）
  │                     └─→ Google Sheets（見積台帳に記録）
  │
  ├─ [FAQ質問] ─→ OpenAI（返信案生成）
  │                   └─→ Gmail（下書き保存）
  │
  ├─ [クレーム] ─→ Slack通知（管理者チャンネル、緊急マーク）
  │                    └─→ HubSpot（チケット作成）
  │
  └─ [その他] ─→ Slack通知（一般チャンネル）
```

**構築手順:**

**Step 1: シナリオの作成**
1. Makeにログイン
2. 「Create a new scenario」をクリック
3. 最初のモジュールとして「Gmail - Watch Emails」を追加

**Step 2: Gmail トリガーの設定**

1. Gmailアカウントを連携
2. フィルタ: 特定のメールアドレス宛（info@, support@等）
3. ポーリング間隔: 15分ごと

**Step 3: OpenAI モジュールの追加**
1. 「OpenAI - Create a Chat Completion」モジュールを追加
2. モデル: gpt-4o
3. プロンプト:

```
以下のメールを分析し、JSON形式で分類してください。

{
  "category": "見積依頼" | "FAQ質問" | "クレーム" | "その他",
  "urgency": "高" | "中" | "低",
  "summary": "メール内容の要約（50字以内）",
  "reply_draft": "FAQ質問の場合のみ、返信案を生成"
}

【件名】{{1.subject}}
【本文】{{1.body}}
```

**Step 4: ルーター（分岐）の追加**
1. 「Router」モジュールを追加
2. 4つのルートを作成し、それぞれにフィルタを設定:
   - ルート1: category が "見積依頼" を含む
   - ルート2: category が "FAQ質問" を含む
   - ルート3: category が "クレーム" を含む
   - ルート4: その他（デフォルト）

**Step 5: 各ルートのアクションを設定**

ルート1（見積依頼）:
1. Slack - Send Message: 営業チャンネルに通知
2. Google Sheets - Add Row: 見積台帳に記録

ルート2（FAQ質問）:
1. OpenAI - Create Chat Completion: 返信案を生成
2. Gmail - Create Draft: 下書きを保存

ルート3（クレーム）:
1. Slack - Send Message: 管理者チャンネルに緊急通知
2. HubSpot - Create Ticket: サポートチケットを作成

**Step 6: エラー処理の追加**
1. 各モジュールに「Error Handler」を追加
2. エラー時のアクション: Slackの管理者チャンネルにエラー通知

### Makeの費用シミュレーション

上記のシナリオで、1日50件のメールを処理する場合:

```
月間のオペレーション数:
  50件/日 x 30日 = 1,500件/月
  1件あたり4オペレーション（Gmail読取 + AI分類 + Slack通知 + 追加処理）
  = 6,000オペレーション/月

必要なプラン: Coreプラン（10,000オペレーション/月）= $9/月
OpenAI API費用: 約$5〜15/月（50件/日 x 30日 = 1,500リクエスト）

合計: 月額約2,100〜3,600円
```


## n8nの実践

### n8nの特徴

n8nはオープンソースのワークフロー自動化ツールだ。自社のサーバーにインストールして使えば、**月額費用が完全に無料**になる。

**n8nが向いている企業:**

- 技術に詳しいIT担当者がいる
- 月額費用を最小限にしたい
- データを外部サービスに預けたくない（セキュリティ要件）
- Zapier/Makeでは対応できないカスタム処理が必要

### セルフホストの手順

**Docker Composeでのインストール:**

```yaml
# docker-compose.yml
version: '3'
services:
  n8n:
    image: n8nio/n8n
    restart: always
    ports:
      - "5678:5678"
    environment:
      - N8N_BASIC_AUTH_ACTIVE=true
      - N8N_BASIC_AUTH_USER=admin
      - N8N_BASIC_AUTH_PASSWORD=your-secure-password
      - GENERIC_TIMEZONE=Asia/Tokyo
    volumes:
      - n8n_data:/home/node/.n8n

volumes:
  n8n_data:
```

```bash
docker-compose up -d
```

これで http://localhost:5678 でn8nにアクセスできる。

**クラウドで運用する場合:**

- AWS EC2（t3.micro: 月約$10）やDigitalOcean Droplet（月$6）にインストール
- n8n Cloud（公式ホスティング: 月$24〜）を利用

### n8nで構築するシナリオ例

**在庫アラート自動化:**

```
Schedule Trigger（毎日9:00実行）
  → Google Sheets（在庫データを読み取り）
  → IF（在庫数 ≦ 発注点）
    → Yes: Slack通知（要発注の商品一覧）
           + Gmail（仕入先に発注メールの下書き作成）
    → No: 何もしない
```

**日次売上レポート自動化:**

```
Schedule Trigger（毎日21:00実行）
  → freee API（当日の売上データ取得）
  → Code Node（JavaScript で集計処理）
  → OpenAI（レポート文面を生成）
  → Slack（#daily-report チャンネルに投稿）
  → Google Sheets（日次データを追記）
```


## 実践的なワークフロー設計のコツ

### コツ1: 「1ワークフロー = 1目的」の原則

1つのワークフローに複数の目的を詰め込まない。「問い合わせ対応の自動化」と「日次レポートの自動化」は別のワークフローにする。これにより、エラーの特定が容易になり、メンテナンス性も向上する。

### コツ2: エラー処理を必ず設定する

ワークフローは24時間自動で動くため、エラーが発生しても気付かないことがある。以下のエラー処理を必ず設定する。

1. **リトライ**: APIの一時的なエラー（タイムアウト等）に対して、自動で再試行する
2. **通知**: エラー発生時にSlackやメールで担当者に通知する
3. **フォールバック**: メインの処理が失敗した場合の代替処理を用意する

### コツ3: テストデータで十分にテストする

本番のデータで直接テストせず、テスト用のデータ（ダミーの顧客情報、テスト用のメールアドレス等）を使ってテストする。特に、メール送信やCRM登録を含むワークフローは、テストデータで動作確認してから本番に切り替える。

### コツ4: ドキュメントを残す

ワークフローの設計者が退職したり異動したりしても運用できるよう、以下をドキュメント化しておく。

- ワークフローの目的と概要
- トリガー条件
- 使用しているツールとアカウント情報（パスワードは別管理）
- 各ステップの処理内容
- エラー時の対応手順


## Power Automate -- Microsoft 365ユーザーのための選択肢

Microsoft 365を使っている企業には、**Power Automate**が最適な選択肢だ。

### Power Automateの利点

1. **Microsoft 365に含まれている**: 追加費用なしで基本機能が使える
2. **Outlookとの連携が強力**: メールの自動処理に最適
3. **SharePointとの統合**: ファイル管理の自動化
4. **Teamsとの連携**: 通知やボットの作成
5. **Power Automate Desktop**: PCの画面操作の自動化（RPA）が無料で使える

### Power Automateで構築できるフロー例

**承認ワークフロー:**
```
社員がSharePointで休暇申請を提出
  → 上長にTeamsで承認依頼通知
  → 上長が承認/却下
  → 結果を申請者にメール通知
  → 承認済みの場合、カレンダーに自動登録
```

**経費レポートの自動化:**
```
Outlookで領収書のメールを受信（件名に「領収書」を含む）
  → 添付ファイルをSharePointの指定フォルダに自動保存
  → Excelの経費台帳に自動追記
  → 月末にTeamsで経費サマリを自動投稿
```

## まとめ

本章のポイントを整理する。

1. **ワークフロー自動化は「トリガー + アクション」のシンプルな仕組みで、ツール間のデータ連携を自動化する**
2. **Zapierはシンプルさ、Makeはコスパと高機能、n8nは無料とカスタマイズ性が強み**
3. **Microsoft 365環境ならPower Automateが最適**
4. **エラー処理とドキュメント化を怠ると、後で大きな問題になる**
5. **月額2,000〜10,000円の投資で、ツール間の手作業を大幅に削減できる**
6. **まずは1つのシンプルなワークフローから始め、段階的に拡張する**
