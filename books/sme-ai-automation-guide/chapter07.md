# 在庫管理・需要予測のAI化 -- 小売・飲食・物流向け

## はじめに -- 在庫は「多すぎても少なすぎても」コストになる

在庫管理は、小売業、飲食業、物流業にとって経営の根幹をなす業務だ。

**過剰在庫のコスト:**
- 保管スペースの賃料
- 商品の劣化・廃棄（飲食業では食品ロス）
- 資金の固定化（在庫に変わったお金は使えない）

**欠品のコスト:**
- 販売機会の損失（売れるはずだった利益を失う）
- 顧客満足度の低下（「いつも品切れ」は客離れの原因）
- 緊急発注による追加コスト（通常より高い価格での仕入れ）

中小企業では、この在庫管理を「ベテランの勘」に頼っているケースが多い。「先週の日曜は雨で売れなかったから、今週は少なめに発注しよう」という判断だ。

しかし、ベテランの勘には限界がある。季節変動、曜日効果、天候、イベント、競合の動向、さらにはSNSでのバズなど、考慮すべき変数が多すぎて、人間の直感だけでは最適解を出し続けることは難しい。

本章では、AIを活用した在庫管理と需要予測の具体的な導入方法を、業界別に解説する。


## 在庫管理の3つのレベル

在庫管理のAI化は、以下の3つのレベルで段階的に進める。

| レベル | 内容 | 難易度 | 効果 |
|--------|------|--------|------|
| レベル1 | 在庫データのデジタル化と可視化 | 低 | 現状把握の精度向上 |
| レベル2 | 発注点管理の自動化 | 中 | 欠品と過剰在庫の防止 |
| レベル3 | AI需要予測に基づく発注最適化 | 中〜高 | 利益の最大化 |


## レベル1: 在庫データのデジタル化と可視化

### なぜデジタル化が最初のステップなのか

AIによる需要予測は、**過去のデータがなければ機能しない**。「去年の同じ時期にどれくらい売れたか」「天候によってどう変動するか」というパターンを学習するためには、最低でも3〜6ヶ月分のデジタルデータが必要だ。

まだ在庫管理が紙やExcelベースの場合は、まずデジタル化から始める。

### POSレジの導入（小売業・飲食業向け）

POSレジ（Point of Sale）は、販売データをリアルタイムで記録するシステムだ。2026年現在、タブレット型のクラウドPOSが主流であり、月額数千円から導入できる。

**主なクラウドPOSレジ:**

| サービス | 月額費用 | 特徴 |
|---------|---------|------|
| スマレジ | 無料〜5,500円 | 在庫管理機能が充実。複数店舗対応 |
| Airレジ | 無料 | リクルート運営。シンプルで使いやすい |
| Square POS | 無料 | 決済と一体化。初期費用が低い |
| ユビレジ | 6,900円〜 | 飲食店特化。メニュー管理が得意 |

**POSレジが提供するデータ:**
- 商品別の販売数量（日次、時間帯別）
- 売上金額（商品別、カテゴリ別）
- 客数、客単価
- 曜日別、時間帯別の売上パターン

### バーコード/QRコード管理の導入

小売業や倉庫業では、商品にバーコードやQRコードを貼付し、スキャンで入出庫を記録する方法が効果的だ。

**手順:**
1. 商品マスタを作成する（商品名、SKU、JANコード等）
2. バーコードラベルを印刷する（市販のラベルプリンタ: 1〜3万円）
3. スマホアプリまたはハンディスキャナで読み取る
4. スキャンデータがクラウドの在庫管理システムに自動反映される

**おすすめの在庫管理ツール:**

| ツール | 月額費用 | 特徴 |
|--------|---------|------|
| ロジクラ | 無料〜9,000円 | ECとの在庫連携が得意 |
| zaico | 無料〜3,980円 | シンプル。スマホ完結 |
| NEXT ENGINE | 10,000円〜 | EC多店舗の在庫一元管理 |

### Googleスプレッドシートでの簡易在庫管理

ツール導入の前段階として、Googleスプレッドシートで在庫管理を始める方法もある。

**スプレッドシートのテンプレート構成:**

```
シート1: 商品マスタ
| SKU | 商品名 | カテゴリ | 仕入単価 | 販売単価 | 発注点 | 発注単位 | リードタイム |

シート2: 入出庫記録
| 日付 | SKU | 入庫/出庫 | 数量 | 備考 |

シート3: 在庫一覧（自動計算）
| SKU | 商品名 | 現在庫数 | 発注点 | ステータス（要発注/OK） |

シート4: 日別販売データ（POS連携またはCSVインポート）
| 日付 | SKU | 販売数量 | 売上金額 |
```

スプレッドシートの関数（SUMIFS、VLOOKUP等）を使えば、在庫一覧の自動計算や、発注点を下回った商品のアラートが実現できる。


## レベル2: 発注点管理の自動化

### 発注点とは

発注点（Reorder Point）とは、「在庫がこの数量まで減ったら発注する」という基準値だ。

**発注点の計算式:**
```
発注点 = 1日あたりの平均販売数量 x リードタイム(日) + 安全在庫
```

**例:**
- 1日あたりの平均販売数量: 10個
- 仕入先からの納品リードタイム: 3日
- 安全在庫: 5個（予想外の需要増に備える）
- 発注点 = 10 x 3 + 5 = **35個**

つまり、在庫が35個を切ったら発注する、というルールだ。

### 自動アラートの設定

在庫管理ツールを使っている場合、発注点を設定すれば在庫が基準値を下回った際に自動通知が送られる。

ツールを使わずスプレッドシートで管理している場合は、Google Apps Script（GAS）で自動アラートを設定できる。

**Google Apps Scriptの例:**

```javascript
function checkInventory() {
  const sheet = SpreadsheetApp.getActiveSpreadsheet()
    .getSheetByName('在庫一覧');
  const data = sheet.getDataRange().getValues();
  const alerts = [];

  // ヘッダー行を除き、発注点を下回った商品をチェック
  for (let i = 1; i < data.length; i++) {
    const sku = data[i][0];
    const name = data[i][1];
    const currentStock = data[i][2];
    const reorderPoint = data[i][3];

    if (currentStock <= reorderPoint) {
      alerts.push(`${name}（${sku}）: 現在庫 ${currentStock}個 ≦ 発注点 ${reorderPoint}個`);
    }
  }

  if (alerts.length > 0) {
    const message = `【要発注】以下の商品が発注点を下回りました:\n\n${alerts.join('\n')}`;
    MailApp.sendEmail('担当者@example.com', '在庫アラート', message);
  }
}
```

このスクリプトを「トリガー」で毎朝9時に実行するよう設定すれば、発注が必要な商品が自動でメール通知される。

### Slack通知にする場合

メールの代わりにSlackに通知する場合は、SlackのIncoming Webhookを使う。

## レベル3: AI需要予測に基づく発注最適化

### なぜ「固定の発注点」では不十分なのか

レベル2の発注点管理は、「1日あたりの平均販売数量」を基準にしている。しかし、実際の販売数量は曜日、季節、天候、イベントなどによって大きく変動する。

**例: 傘の販売数（1日あたり）**
- 晴れの平日: 2本
- 晴れの週末: 5本
- 雨の平日: 15本
- 雨の週末: 30本
- 台風予報日: 50本

「平均10本」で発注点を設定しても、天候次第で過剰在庫にも欠品にもなる。AI需要予測は、このような複数の変数を考慮して、日次の需要を予測する。

### ChatGPT/Claudeを使った簡易需要予測

高度な予測AIを導入する前に、まずは生成AIを使った簡易的な需要予測を試してみよう。

**手順:**

1. 過去6〜12ヶ月の日別販売データをCSVで用意する
2. ChatGPT（Plus以上）にCSVをアップロードする
3. 以下のようなプロンプトを入力する

```
アップロードしたCSVは、当店の日別販売データです。

以下の分析を行ってください:
1. 曜日別の平均販売数量をグラフで可視化
2. 月別のトレンド分析
3. 来週（月〜日）の日別販売予測

予測の際は、曜日効果と季節トレンドを考慮してください。
```

ChatGPTはデータを分析し、グラフ付きのレポートと予測値を生成する。この予測値をもとに発注量を決定する。

**精度の目安:** この方法での予測精度は、固定の発注点（平均値ベース）と比較して**15〜25%の改善**が見込める。専用の予測AIには及ばないが、追加費用はChatGPTの月額料金のみ（3,000円）で済む。

### 専用の需要予測サービスを使う場合

より高い精度を求める場合は、専用の需要予測サービスを利用する。

| サービス | 月額費用 | 特徴 |
|---------|---------|------|
| Amazon Forecast | 従量課金（月5,000〜30,000円） | AWSの予測サービス。高精度 |
| Prediction One（Sony） | 要問合せ | プログラミング不要。日本語対応 |
| DataRobot | 要問合せ | AutoMLの先駆け。高機能 |

**中小企業へのお勧め:** まずはChatGPT/Claudeでの簡易予測を3ヶ月試し、効果が確認できたら専用サービスへの移行を検討する。


## 業界別の具体的な導入パターン

### 小売業: EC + 実店舗の在庫一元管理

**課題:** ECサイトと実店舗で在庫データがバラバラ。ECで注文が入ったのに実店舗で売れてしまい、キャンセル対応が月に10件以上発生。

**解決策:**

1. クラウド在庫管理ツール（ロジクラ等）を導入
2. ECサイト（Shopify/BASE/楽天等）と在庫を自動同期
3. 実店舗のPOSレジとも連携
4. 在庫が動くたびにリアルタイムで全チャネルに反映

**構成図:**
```
ECサイト（Shopify）
     ↕  API連携
  クラウド在庫管理（ロジクラ）
     ↕  API連携
実店舗POS（スマレジ）
     ↕
  在庫データベース（一元管理）
     ↓
  在庫数が変動 → 全チャネルに自動反映
```

**効果:**
- 在庫不一致によるキャンセル: 月10件 → 月0〜1件
- 棚卸し作業: 2日 → 半日
- 欠品による機会損失: 月15万円 → 月3万円

### 飲食業: 食品ロス削減のためのAI需要予測

**課題:** 食材の発注量が経験と勘に依存。廃棄食材が月20万円以上発生。

**解決策:**

1. POSデータから日別・メニュー別の販売数を記録
2. 天候データ（気象庁API、無料）を取得
3. イベント情報（近隣のコンサート、スポーツ試合等）を考慮
4. 以上のデータから、生成AIまたは専用ツールで翌日〜3日後の需要を予測
5. 予測に基づいて食材の発注量を決定

**生成AIでの予測プロンプト例:**

```
以下は当店の過去3ヶ月の日別売上データです。

【データ項目】
- 日付、曜日
- 天候（晴/曇/雨）
- 来店客数
- メニューA〜Eの販売数

来週の月〜日について、以下の条件で各メニューの需要を予測してください:
- 月曜: 晴れ
- 火曜: 曇り
- 水曜: 雨（近隣でコンサートあり）
- 木曜〜日曜: 晴れ

各メニューの予測販売数と、食材の推奨発注量を算出してください。
食材の最小発注単位は以下の通りです:
- 鶏肉: 1kg単位（メニューA 1食あたり200g使用）
- 野菜セット: 1箱単位（メニューB,C各1食あたり1/4箱使用）
...
```

**効果:**
- 食品廃棄額: 月20万円 → 月7万円（年間156万円の改善）
- 欠品（品切れ）: 月15回 → 月3回
- 発注作業時間: 45分/日 → 15分/日

### 物流業: 配送ルート最適化と在庫配置

**課題:** 複数の倉庫・配送拠点間で在庫の偏りが発生。A倉庫では在庫過多、B倉庫では欠品、という状態が日常的に起きている。

**解決策:**

1. 全倉庫の在庫データをクラウドで一元管理
2. 配送先の地域別需要を分析
3. 在庫の最適配置（どの倉庫にどれだけ置くか）をAIが算出
4. 倉庫間の在庫移動を自動提案

**効果:**
- 倉庫間の緊急在庫移動: 月15回 → 月3回
- 配送コスト: 10〜15%削減（倉庫と配送先の距離が短縮）
- 欠品による緊急配送: 月8回 → 月1回


## 導入スケジュール（全業界共通）

| 期間 | やること | ゴール |
|------|---------|--------|

| 3ヶ月目 | 生成AIでの簡易需要予測を開始 | 発注精度の向上 |
| 4〜6ヶ月目 | 予測精度の評価と改善 | 安定運用 |

## まとめ

本章のポイントを整理する。

1. **在庫管理のAI化は「デジタル化 → 発注点管理 → 需要予測」の3段階で進める**
2. **まずデータを蓄積することが最優先。AI予測にはデータが必要**
3. **発注点管理の自動化だけでも、欠品と過剰在庫は大幅に改善する**
4. **ChatGPT/Claudeでの簡易需要予測で、発注精度を15〜25%改善できる**
5. **飲食業では食品ロス削減により年間100万円以上の改善が見込める**
6. **小売業ではEC+実店舗の在庫一元管理が最優先課題**

次章では、日報・議事録・ドキュメント作成の自動化を解説する。全社員が関わる業務だけに、自動化の効果は組織全体に波及する。
