# 日報・議事録・ドキュメント作成の自動化

## はじめに -- 「書く」仕事は全社員に関わるからこそ、効果が大きい

経理の自動化は経理担当者にしか恩恵がない。営業の自動化は営業チームにしか関係ない。しかし、ドキュメント作成の自動化は**全社員に恩恵がある**。

なぜなら、日報、議事録、報告書、マニュアル、提案書。これらの「書く作業」は、役職や部門を問わず、すべてのビジネスパーソンが日常的に行っているからだ。

試算してみよう。従業員30名の企業で、1人あたり「書く作業」に1日30分を使っているとする。

```
30名 x 0.5時間/日 x 20営業日/月 = 月間300時間
```

この300時間のうち、50%を自動化できれば月間150時間の創出。時給3,000円で換算すると**月間45万円、年間540万円**の価値がある。

本章では、中小企業で最も頻度の高い4つのドキュメントについて、自動化の方法を解説する。

```
自動化対象:
  1. 日報 -- 毎日、全社員が作成
  2. 議事録 -- 会議のたびに作成
  3. 報告書・レポート -- 週次・月次で作成
  4. マニュアル・手順書 -- 業務変更のたびに更新
```


## 1. 日報の自動化

### 現状の課題

「日報を書くのが面倒」-- これは全国の中小企業の社員が共感する声だろう。

日報の目的は「今日やったことの記録」と「上司への報告」だ。しかし実態としては、以下の問題を抱えていることが多い。

- 退勤間際に急いで書くため、内容が薄い
- フォーマットが統一されておらず、上司が読むのも大変
- 書いても誰もフィードバックしないので、モチベーションが下がる
- 蓄積されたデータが活用されていない

### 自動化のアプローチ

日報の自動化には3つのアプローチがある。

**アプローチ1: 音声入力 + AI要約**

最もハードルが低い方法だ。

1. 退勤前に、スマホの音声メモ機能で「今日やったこと」を2〜3分話す
2. 音声データをテキスト化する（Whisper API、またはスマホの音声入力機能）
3. テキストを生成AIに送り、日報フォーマットに整形させる

**プロンプト例:**
```
以下は営業担当者の音声メモのテキスト化です。
これを日報フォーマットに整形してください。

【フォーマット】
■ 今日の業務内容
  - （箇条書きで3〜5項目）
■ 進捗・成果
  - （数値があれば含める）
■ 課題・相談事項
  - （あれば記載）
■ 明日の予定
  - （箇条書きで2〜3項目）

【音声メモのテキスト】
えーと今日は朝一で田中商事さんに訪問して、新しい提案書を持っていきました。
先方の部長さんは好感触で、来週中に社内稟議にかけてくれるとのこと。
午後は社内でCRM入力と、あと山田さんの案件の見積を手伝いました。
明日は午前中にABC株式会社さんとオンラインミーティング、午後は新規開拓の電話営業の予定です。
あ、あと田中商事さんの件で、値引きの相談が来そうなので、上限を確認しておきたいです。
```

**AI出力例:**
```
■ 今日の業務内容
  - 田中商事 訪問（新提案書の提出）
  - CRMデータ入力
  - 山田案件の見積作成支援

■ 進捗・成果
  - 田中商事: 部長が好感触、来週中に社内稟議予定

■ 課題・相談事項
  - 田中商事から値引き相談の可能性あり → 値引き上限の確認が必要

■ 明日の予定
  - 午前: ABC株式会社 オンラインミーティング
  - 午後: 新規開拓の電話営業
```

2〜3分の音声メモが、整理された日報に変換される。手書きなら15分かかる作業が3分で終わる。

**アプローチ2: ツール連携による自動生成**

Googleカレンダー、Slack、CRM、プロジェクト管理ツールなどの利用ログから、日報を自動生成する方法だ。

**構成例:**
```
データソース:
  - Googleカレンダー → 今日参加したミーティング一覧
  - Slack → 投稿したメッセージの要約
  - HubSpot CRM → 更新した取引情報
  - GitHub → プッシュしたコード（エンジニアの場合）

自動生成フロー（Zapier/Makeで構築）:
  毎日18:00に以下を実行
    1. Googleカレンダーから今日の予定を取得
    2. Slackの投稿内容を取得
    3. CRMの更新履歴を取得
    4. これらを生成AIに送り、日報フォーマットで出力
    5. 結果をSlackの日報チャンネルに投稿（または上司にメール送信）
```

この方法なら、社員は日報を「書く」必要すらない。ツールを普通に使っているだけで、日報が自動生成される。

**アプローチ3: チャットボット形式**

Slackボットが退勤前に質問を投げかけ、回答をもとに日報を自動生成する。

```
ボット: 今日お疲れ様でした。日報の時間です。
ボット: 今日の主な業務を教えてください。
社員: 田中商事に訪問、午後はCRM入力
ボット: 進捗や成果はありましたか？
社員: 田中商事の部長が好感触、来週稟議
ボット: 明日の予定は？
社員: ABC社とオンラインミーティング
ボット: ありがとうございます。日報を生成しました。

【自動生成された日報を表示】

ボット: こちらで問題なければ「OK」、修正があれば内容を教えてください。
社員: OK
ボット: 日報を投稿しました。
```

対話形式なので、心理的な負担が「日報を書く」よりも格段に低い。

### 導入の推奨

| 企業の状況 | 推奨アプローチ |
|----------|-------------|
| まず手軽に始めたい | アプローチ1（音声入力+AI） |
| すでにSlack等を導入済み | アプローチ3（チャットボット形式） |
| Googleカレンダー+CRMを使っている | アプローチ2（ツール連携自動生成） |


## 2. 議事録の自動化

### 現状の課題

会議の議事録は「誰がとるか」でいつも揉める業務の代表だ。若手が押し付けられることが多いが、メモを取ることに集中すると議論に参加できなくなる。

さらに、手動で取った議事録には以下の問題がある。
- メモの取り漏れ（発言のすべてを記録できない）
- 主観が入る（記録者のフィルターがかかる）
- 清書に時間がかかる（30分の会議に対して、議事録作成に30〜60分）

### 自動化のフロー

```
会議中: 音声を自動でテキスト化（文字起こし）
    ↓
会議後: 生成AIが議事録を自動生成
    ↓
確認: 参加者が内容を確認・修正
    ↓
共有: 関係者に自動配信
```

### 手順1: 会議音声のテキスト化

**オンライン会議の場合（Zoom/Google Meet/Teams）:**

各ツールに標準搭載されている文字起こし機能を使う。

| ツール | 文字起こし機能 | 日本語精度 |
|--------|-------------|----------|
| Zoom | 標準搭載（Pro以上） | 良好 |
| Google Meet | 標準搭載（Workspace） | 良好 |
| Microsoft Teams | 標準搭載 | 良好（Copilot連携で要約も可能） |

Zoomの場合:
1. ミーティング設定で「自動字幕（文字起こし）」を有効にする
2. 会議が終了すると、テキストファイルがダウンロード可能になる

**対面会議の場合:**

スマホやICレコーダーで録音し、文字起こしサービスで変換する。

| サービス | 費用 | 特徴 |
|---------|------|------|
| CLOVA Note | 無料（月300分まで） | LINE提供。日本語精度が高い |
| Notta | 無料〜1,317円/月 | リアルタイム文字起こし対応 |
| otter.ai | 無料〜$16.99/月 | 英語が非常に高精度 |
| Amazon Transcribe | 従量課金 | AWS。多言語対応 |

**おすすめ:** 日本語の対面会議なら、CLOVA Noteが無料かつ高精度で始めやすい。

### 手順2: 生成AIによる議事録の自動生成

文字起こしテキストを生成AIに入力し、構造化された議事録に変換する。

**プロンプト例:**
```
以下は社内ミーティングの文字起こしテキストです。
これを議事録フォーマットに整形してください。

【議事録フォーマット】
会議名:
日時:
参加者:
---
■ アジェンダ
  1.（議題を箇条書き）

■ 議論の要約
  - （議題ごとに要点を3〜5行で）

■ 決定事項
  - （何が決まったかを明確に）

■ TODO（アクションアイテム）
  - [ ] 担当者: タスク内容 （期限: ○月○日）

■ 次回の予定
  - 日時、議題

【文字起こしテキスト】
（ここに文字起こしテキストを貼り付け）
```

**コツ:** 文字起こしテキストが長い場合（1時間以上の会議）は、Claude のように長文入力に対応したAIを使うとよい。Claude Proは1回の入力で数万字のテキストを処理できる。

### 手順3: 自動配信

生成された議事録を、参加者に自動で配信する。

**Zapier/Makeでの自動化例:**
```
Googleドライブに議事録ファイルが保存される
  → Zapierが検知
  → Slackの該当チャンネルに投稿
  → 参加者にメールで送信
```

### 効果の目安

| 指標 | 手動 | 自動化後 |
|------|------|---------|
| 議事録の作成時間 | 30〜60分/回 | 5〜10分/回（確認・修正のみ） |
| 記録の完全性 | 60〜70%（取り漏れあり） | 95%以上 |
| 議事録の配信速度 | 翌日〜3日後 | 会議終了後30分以内 |
| 議事録担当の心理的負担 | 高い | ほぼゼロ |


## 3. 報告書・レポートの自動化

### 週次報告書の自動生成

多くの中小企業では、週次の進捗報告が求められる。プロジェクト管理ツール（Notion、Asana、Trello、Backlog等）を使っていれば、そのデータから報告書を自動生成できる。

**構成例（Notionを使っている場合）:**

```
データソース:
  Notionのタスクデータベース
    → 今週完了したタスク
    → 進行中のタスク
    → 期限超過のタスク

自動生成フロー:
  毎週金曜17:00に実行
    1. Notion APIからタスクデータを取得
    2. 生成AIで週次報告書を作成
    3. PDFまたはMarkdownで出力
    4. 上司にメール/Slack送信
```

**報告書テンプレートのプロンプト:**
```
以下のタスクデータから、週次進捗報告書を作成してください。

【フォーマット】
# 週次進捗報告書（○月○日〜○月○日）

## 今週の成果
- （完了タスクの概要。ビジネスインパクトを含める）

## 進行中の案件
| 案件名 | 進捗率 | 予定完了日 | 備考 |

## 課題・リスク
- （期限超過のタスクや、リスクのある案件）

## 来週の計画
- （優先度の高いタスク3〜5件）

【今週のタスクデータ】
（Notion APIから取得したデータを挿入）
```

### 月次レポートの自動生成

経営者向けの月次レポートも、データソースが整っていれば自動生成が可能だ。

**データソース例:**

- freee/マネーフォワードから売上・経費データ
- Google Analyticsからアクセスデータ
- HubSpot CRMから営業データ
- POSレジから販売データ

これらのデータをスプレッドシートに集約し、生成AIがレポートに整形する。

**効果:**

- 月次レポート作成: 4〜6時間/月 → 30分/月（確認・修正のみ）


## 4. マニュアル・手順書の自動化

### 現状の課題

社内マニュアルは「作るのは大変だが、作らないともっと大変」な存在だ。

特に中小企業では、業務手順がベテラン社員の頭の中にだけ存在し、文書化されていないことが多い。その結果、ベテランが休んだり退職したりすると、業務が止まる。

### 生成AIを活用したマニュアル作成

**方法1: 画面録画から手順書を自動生成**

PC操作の手順書を作る場合、画面録画ツールで操作を録画し、そのスクリーンショットと操作ログから手順書を自動生成する。

手順:
1. 画面録画ツール（Loom、OBS等）で操作を録画する
2. 録画を見ながら、操作の要所でスクリーンショットを取る
3. スクリーンショットと操作の説明を生成AIに渡す
4. AIがマニュアル形式に整形する

**方法2: ベテラン社員へのインタビューからマニュアルを生成**

ベテラン社員に業務の手順を口頭で説明してもらい、その音声を文字起こし → 生成AIでマニュアル化する。議事録の自動化と同じ仕組みだ。

**プロンプト例:**
```
以下は、経理担当者による月次決算の手順説明の文字起こしです。
これを、経理未経験者でも実行できるレベルの手順書に整形してください。

【手順書のフォーマット】
## 手順書: [業務名]
### 目的
### 前提条件（必要な権限、ツール等）
### 手順
  1. （具体的なステップ。画面の名称やボタン名を明記）
  2. ...
### 注意事項
### トラブルシューティング（よくあるエラーと対処法）

【文字起こしテキスト】
（ベテラン社員のインタビュー内容）
```

**方法3: 既存マニュアルの更新を自動化**

業務フローが変更されたとき、既存のマニュアルを自動的に更新する仕組みを作る。

```
業務フローの変更を記録（変更メモをSlackに投稿）
  → 生成AIが変更内容を解析
  → 既存マニュアルの該当箇所を特定
  → 更新案を生成
  → 担当者が確認・承認
  → マニュアルに反映
```

### マニュアルを「検索可能な知識ベース」にする

作成したマニュアルは、Notion、Confluence、Google Driveなどに蓄積し、**社内チャットボット（第5章で紹介したDify等）のナレッジベース**として活用する。

これにより、社員が「経費精算のやり方がわからない」と質問したとき、AIチャットボットがマニュアルを参照して回答してくれる。マニュアルを「読む」から「質問する」に変わることで、活用率が大幅に向上する。


## ドキュメント自動化の統合プラットフォーム

### Notion AI の活用

Notion AIは、ドキュメント作成の自動化に最も適したツールの1つだ。

**Notion AIでできること:**

- 文章の自動生成（プロンプトを入力するだけ）
- 既存テキストの要約・拡張・翻訳
- テンプレートの自動適用
- データベースからの自動レポート生成
- 議事録の要約
- TODO抽出

**費用:** Notion自体は無料〜月額1,650円/ユーザー。Notion AIは追加で月額1,350円/ユーザー。

### Google Workspace + Gemini の活用

GoogleドキュメントにGeminiが統合されており、ドキュメント作成中にAIの支援を受けられる。

**できること:**

- 「Help me write」機能でドキュメントのドラフトを自動生成
- スプレッドシートのデータから自動でグラフを作成
- スライドの自動デザイン
- Gmailの下書き自動生成

**費用:** Google Workspace Business Standard（月額1,496円/ユーザー）以上で利用可能。

## まとめ

本章のポイントを整理する。

1. **ドキュメント作成の自動化は全社員に恩恵があり、組織全体の生産性向上に直結する**
2. **日報は「音声入力+AI要約」で作成時間を1/5に短縮できる**
3. **議事録は「自動文字起こし+AI整形」で、作成時間を30〜60分から5〜10分に削減**
4. **週次・月次レポートは、ツールからのデータ自動取得+AI整形で、ほぼ全自動化が可能**
5. **マニュアルは「ベテランの暗黙知」をAIで文書化し、社内チャットボットの知識ベースとして活用する**
6. **月額1〜2万円の投資で、全社で月間150時間以上の工数を削減できる**
