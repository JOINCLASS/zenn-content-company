# 経理・請求処理の自動化 -- freee/マネーフォワード活用

## はじめに -- 経理は「最もROIの高い」自動化対象

中小企業のAI業務自動化で、最初に着手すべき領域を1つだけ挙げるなら、私は「経理」と答える。理由は3つある。

1. **定型度が高い**: 仕訳、請求書発行、消込、経費精算は、ルールが明確で自動化しやすい
2. **ミスのコストが大きい**: 金額の間違いは取引先との信頼問題に直結する。自動化で人的ミスを減らせる
3. **効果がすぐ見える**: 「毎月2日かかっていた作業が半日になった」という成果が明確に数字で出る

本章では、中小企業の経理業務を3つの領域に分けて、それぞれの自動化手順を具体的に解説する。

```
経理業務の3つの領域:
  1. 日常仕訳・記帳の自動化
  2. 請求書の発行・送付の自動化
  3. 経費精算の自動化
```


## 前提: クラウド会計ソフトの選び方

経理の自動化は、クラウド会計ソフトの導入から始まる。まだExcelや紙ベースで経理をしている場合は、まずクラウド会計ソフトへの移行が必要だ。

| 銀行口座自動連携 | 3,200行以上 | 3,600行以上 | 主要行対応 |

| 他サービス連携 | freeeシリーズと統合 | MFシリーズと統合 | 弥生シリーズ内 |
| 確定申告対応 | 個人・法人 | 個人・法人 | 個人・法人 |

**筆者のお勧め:** 経理の専任者がいない企業はfreee、経理経験者がいる企業はマネーフォワード。本章では両方の手順を解説する。

### 移行の注意点

ExcelやデスクトップPCの会計ソフトからクラウド会計に移行する際、最も注意すべきは「期首残高の引き継ぎ」だ。

具体的な手順は以下の通り。

1. 移行前のソフトで、直前の決算時点の貸借対照表を出力する
2. クラウド会計ソフトに期首残高として登録する
3. 移行日以降の取引は、クラウド会計ソフトに入力する
4. 移行初月は、旧ソフトと新ソフトの両方に記帳し、数字が一致するか確認する（並行稼働）

並行稼働は面倒だが、1ヶ月だけ我慢すれば、その後は自動化の恩恵を長期間受けられる。


## 領域1: 日常仕訳・記帳の自動化

### 現状の課題

多くの中小企業では、以下のような手作業が毎日発生している。

1. ネットバンキングにログインして、前日の入出金を確認する
2. 各取引の内容を確認し、勘定科目を判断する
3. 会計ソフト（またはExcel）に仕訳を入力する
4. 通帳やレシートと突合して、漏れがないか確認する

この作業は、取引先が多い企業では1日30分〜1時間、月間で10〜20時間に達する。

### 自動化の全体像

```
自動化前:
  ネットバンキング確認 → 仕訳判断 → 手入力 → 突合確認
  （すべて人間の作業）

自動化後:
  銀行API自動取込 → AI仕訳推測 → 人間が確認・承認 → 自動記帳
  （人間は「確認・承認」のみ）
```

### 手順1: 銀行口座の自動連携を設定する

**freeeの場合:**

1. freee会計にログインする
2. 「口座」→「口座を登録する」をクリック
3. 「銀行・カード等を登録」から、使用している金融機関を選択する
4. インターネットバンキングのID・パスワードを入力して連携する
5. 連携が完了すると、毎日自動で入出金データが取り込まれる

**マネーフォワードの場合:**

1. マネーフォワードクラウド会計にログインする
2. 「データ連携」→「新規登録」をクリック
3. 金融機関を検索して選択する
4. ログイン情報を入力して連携する
5. 初回は過去の取引も取り込まれる（最大2ヶ月分）

**連携可能な金融機関の例:**
メガバンク（三菱UFJ、三井住友、みずほ）、ゆうちょ銀行、楽天銀行、住信SBIネット銀行、PayPay銀行、各地方銀行、信用金庫など。法人口座にも対応している。

**クレジットカードも連携する:**
法人カードや個人カード（経費で使用するもの）も連携しておくと、カード明細も自動取込される。経費精算の手間が大幅に減る。

### 手順2: AI仕訳推測のルールを育てる

銀行口座を連携すると、取り込まれた取引データに対して、AIが勘定科目の候補を自動推測してくれる。

**最初は精度が低い。** これは当然だ。AIはまだあなたの会社の取引パターンを知らない。しかし、人間が正しい勘定科目を選んで承認するたびに、AIは学習する。2〜3ヶ月で精度は90%以上に達する。

**学習を加速させるコツ:**

1. **自動仕訳ルールを手動で設定する**: 「取引先名に"Amazon"が含まれる → 消耗品費」のようなルールを事前に設定しておく
2. **摘要を統一する**: 振込時に摘要欄に書く文字列を社内で統一する（例: 「給料_2月分」「家賃_2月」）
3. **頻出取引から先に対応する**: 月に1回しか発生しない取引よりも、毎日発生する取引を先にルール化する

**freeeでの自動仕訳ルール設定手順:**

1. 「自動で経理」画面を開く
2. 未処理の取引の中から、ルール化したいものを選ぶ
3. 正しい勘定科目を選択し、「自動登録ルールとして保存」にチェックを入れる
4. ルールの条件（取引先名、金額範囲等）を確認して保存する
5. 次回以降、同じパターンの取引は自動で仕訳される

### 手順3: 日次の確認フローを確立する

自動化後も、人間による確認は必要だ。ただし、確認にかかる時間は大幅に短縮される。

**毎日の確認フロー（所要時間: 5〜10分）:**

1. クラウド会計ソフトの「未処理取引」を開く
2. AIが推測した仕訳を確認する（金額、勘定科目、取引先）
3. 正しければ「承認」、間違っていれば修正して「承認」
4. 不明な取引は「保留」にして、後で確認する

**コツ:** この確認作業は、毎朝の最初の5分で行うようにルーティン化するとよい。溜まると面倒になるが、毎日やれば数分で終わる。

### 効果の目安

| 項目 | 自動化前 | 自動化後 |
|------|---------|---------|
| 日次の仕訳作業 | 30〜60分/日 | 5〜10分/日 |
| 月間の仕訳工数 | 10〜20時間 | 2〜4時間 |
| 仕訳ミスの発生率 | 月3〜5件 | 月0〜1件 |
| 月次決算の速度 | 翌月15日頃 | 翌月5日頃 |


## 領域2: 請求書の発行・送付の自動化

### 現状の課題

請求書の発行と送付は、多くの中小企業で以下のような手作業で行われている。

1. Excelのテンプレートを開き、取引先名、金額、品目を入力する
2. PDFに変換する
3. メールに添付して送付する
4. 送付済みの管理台帳に記録する
5. 入金を確認したら、消込処理を行う

取引先が20社あれば、この作業だけで月に丸1日以上かかる。しかも、インボイス制度により、請求書の記載要件が増えているため、手作業での対応はますます難しくなっている。

### 自動化の全体像

```
自動化前:
  Excel作成 → PDF変換 → メール添付送付 → 台帳記録 → 入金確認 → 消込
  （すべて人間の作業。取引先ごとに繰り返し）

自動化後:
  取引データから一括生成 → ワンクリックで一括送付 → 入金自動マッチング → 自動消込
  （人間は「確認と送信ボタン」のみ）
```

### 手順1: 請求書テンプレートを設定する

**freeeの場合:**

1. 「設定」→「請求書設定」を開く
2. 会社名、住所、振込先口座を登録する
3. 適格請求書発行事業者の登録番号を設定する（インボイス対応に必須）
4. ロゴ画像をアップロードする
5. 通常使う税率（10%、8%等）を設定する
6. 請求書番号の採番ルールを設定する（例: INV-2026-0001）

**マネーフォワードの場合:**

1. 「請求書」→「設定」を開く
2. 事業者情報を登録する
3. テンプレートのデザインを選ぶ（複数のテンプレートから選択可能）
4. デフォルトの支払条件（例: 月末締め翌月末払い）を設定する
5. 適格請求書の要件に沿った項目が自動で含まれるか確認する

### 手順2: 取引先マスタを登録する

請求書を毎月発行する取引先の情報を事前に登録しておく。

登録する情報:
- 取引先名
- 担当者名とメールアドレス（請求書送付先）
- 締め日と支払日（例: 月末締め翌月末払い）
- 消費税の計算方法（税込/税抜）
- 源泉徴収の有無

**この初期設定は1回だけ。** 取引先が20社でも、1〜2時間で完了する。

### 手順3: 請求書の一括生成と送付

**freeeでの月次請求フロー:**

1. 「請求書」→「新規作成」をクリック
2. 取引先を選択すると、前月の請求書をコピーして新規作成できる
3. 金額や品目を今月分に修正する
4. 「プレビュー」で内容を確認する
5. 「メール送付」で取引先に直接送信する（PDFが自動添付される）
6. 送付済みの請求書は自動で売掛金として仕訳される

**さらに効率化: 定期請求の自動化**

毎月同じ金額の請求（月額のサービス料、家賃の代行請求等）がある場合、「定期請求」機能で自動化できる。

1. 「定期請求」を設定する（取引先、金額、品目、発行日を指定）
2. 毎月指定日に請求書が自動生成される
3. 自動送付を設定すれば、メール送信まで自動で完了する

### 手順4: 入金消込の自動化

請求書を送ったら、入金を確認して消込（売掛金を回収済みにする処理）を行う必要がある。

**自動消込の仕組み:**

銀行口座を連携している場合、入金データが自動で取り込まれる。freeeやマネーフォワードのAIは、入金額と請求書の金額を自動でマッチングし、消込候補を提示してくれる。

人間は、マッチングが正しいかを確認して「承認」するだけだ。

**マッチングの精度を上げるコツ:**
- 請求書に振込依頼人名を指定しておく（「カ）○○商事」のように）
- 端数のない金額にする（110,000円は自動マッチングしやすいが、109,780円は候補が複数出やすい）
- 合計払いの場合は、請求書をまとめて1枚にする

### 手順5: 入金遅延のアラート設定

支払期日を過ぎても入金がない場合、自動でアラートが出るように設定する。

**freeeの場合:**
1. 「レポート」→「売掛レポート」を開く
2. 「入金期日を過ぎた未回収の請求書」を表示する
3. 該当する請求書に対して、催促メールのテンプレートを使って連絡する

**ワークフロー自動化との連携:**
Zapier や Make を使えば、「支払期日を3日過ぎても入金がない場合 → 担当者にSlack通知」という自動化も可能だ。第10章で詳しく解説する。

### 効果の目安

| 項目 | 自動化前 | 自動化後 |
|------|---------|---------|
| 請求書作成（20社分） | 6〜8時間/月 | 1〜2時間/月 |
| 請求書送付 | 2時間/月 | 10分/月（一括送信） |
| 入金消込 | 3〜4時間/月 | 30分/月 |
| 未回収管理 | 1時間/月 | 自動通知 |
| **合計** | **12〜15時間/月** | **2〜3時間/月** |


## 領域3: 経費精算の自動化

### 現状の課題

経費精算は、多くの中小企業で「最も面倒な業務」の1つだ。

典型的なフロー:
1. 社員がレシートを集めて、Excelの経費精算書に記入する
2. 領収書をノリで台紙に貼り付ける
3. 上長に承認を求める（紙の回覧、またはメール）
4. 経理がExcelの内容を確認し、会計ソフトに仕訳を入力する
5. 振込処理を行う

このフローの問題点は明白だ。紙のレシートが紛失する、記入ミスがある、承認に時間がかかる、経理の二重入力が発生する。

### 自動化の全体像

```
自動化前:
  レシート収集 → Excel記入 → 紙に貼付 → 上長承認（紙回覧） → 経理入力 → 振込
  （全ステップが手作業）

自動化後:
  スマホ撮影 → OCR自動読取 → アプリで申請 → アプリで承認 → 自動仕訳 → 自動振込
  （人間は「撮影」「確認」「承認ボタン」のみ）
```

### 手順1: 経費精算ツールを導入する

freeeやマネーフォワードには、経費精算機能が含まれている（または別サービスとして提供されている）。

**主な経費精算ツール:**

| ツール | 月額費用 | 特徴 |
|--------|---------|------|
| freee経費精算 | freee会計に含まれる | freeeユーザーなら追加費用なし |
| マネーフォワードクラウド経費 | 500円/ユーザー〜 | 領収書OCRの精度が高い |
| 楽楽精算 | 要問合せ | 大企業での導入実績が豊富 |
| TOKIUM経費精算 | 要問合せ | 領収書を郵送するだけでデータ化してくれる |

**freee会計を使っている場合:** 追加のツール導入は不要。freeeのスマホアプリで経費精算が完結する。

### 手順2: レシートのOCR読取を設定する

**OCR（光学文字認識）** とは、カメラで撮影した画像からテキスト情報を自動で読み取る技術だ。

**freeeアプリでの操作:**
1. スマホアプリを開く
2. 「レシート撮影」をタップ
3. レシートをカメラで撮影する
4. AIが自動で「日付」「金額」「取引先名」を読み取る
5. 読み取り結果を確認し、間違いがあれば修正する
6. 勘定科目を選択する（AIが候補を提示してくれる）
7. 「申請」ボタンを押す

**精度の実績:** 最新のOCR技術では、レシートの読取精度は95%以上。ただし、手書きの領収書やかすれたレシートは精度が下がるため、撮影時に鮮明に撮ることが重要だ。

### 手順3: 承認フローをデジタル化する

紙の回覧やメールでの承認を、アプリ内の承認フローに変更する。

**設定手順:**
1. 承認者（上長）を設定する
2. 金額に応じた承認ルートを設定する
   - 1万円未満: 部長承認のみ
   - 1万円以上5万円未満: 部長承認 → 経理確認
   - 5万円以上: 部長承認 → 社長承認 → 経理確認
3. 社員が経費を申請すると、承認者にプッシュ通知が届く
4. 承認者はスマホで内容を確認し、「承認」または「差戻し」をタップする

**承認のスピードアップ効果:** 紙の回覧では承認に平均3日かかっていたものが、アプリなら平均4時間に短縮される（承認者がプッシュ通知に気付いてすぐ対応する場合）。

### 手順4: 法人カードの活用で経費精算自体を減らす

ここまでは「経費精算の効率化」を解説したが、最も効果的な方法は**「経費精算の発生自体を減らす」**ことだ。

法人カード（コーポレートカード）を導入し、経費の支払いをカードに集約すれば、立替精算が不要になる。

**法人カードのメリット:**
1. 立替と精算の手間がなくなる
2. カード明細が自動でクラウド会計に取り込まれる
3. 利用限度額で支出をコントロールできる
4. ポイントが貯まる（年間の経費が500万円なら、1%還元で5万円相当）

**中小企業向け法人カードの例:**
- 三井住友カード ビジネスオーナーズ（年会費無料）
- JCB法人カード（年会費1,375円〜）
- アメリカン・エキスプレス・ビジネス・カード（年会費13,200円。特典が充実）

### 効果の目安

| 項目 | 自動化前 | 自動化後 |
|------|---------|---------|
| 社員の経費精算作業 | 1時間/人/月 | 10分/人/月 |
| 承認所要日数 | 3日 | 4時間 |
| 経理の処理時間 | 4時間/月 | 30分/月 |
| レシート紛失 | 月3〜5件 | 0件（撮影即アップロード） |
| 仕訳ミス | 月2〜3件 | 月0〜1件 |


## 応用: インボイス制度と電子帳簿保存法への対応

2023年10月に開始されたインボイス制度と、2024年1月に義務化された電子帳簿保存法。この2つの法制度への対応は、中小企業の経理担当者にとって大きな負担になっている。

しかし、クラウド会計ソフトを使っていれば、**これらの法制度への対応はほぼ自動化されている。**

### インボイス制度への対応

**必要な対応:**
- 発行する請求書に「適格請求書発行事業者の登録番号」を記載する
- 受け取った請求書が適格請求書の要件を満たしているか確認する
- 税率ごとに区分して記帳する

**クラウド会計ソフトでの自動対応:**
- 請求書テンプレートに登録番号が自動挿入される
- 受取請求書のOCRで登録番号を自動読取し、国税庁のデータベースと自動照合する
- 税率区分は仕訳時に自動設定される

### 電子帳簿保存法への対応

**必要な対応:**
- 電子取引データ（メールで受け取った請求書のPDF等）を、検索可能な形式で保存する
- タイムスタンプを付与する
- 7年間保存する

**クラウド会計ソフトでの自動対応:**
- アップロードしたPDFやスキャンデータに自動でタイムスタンプが付与される
- 取引日、金額、取引先名で検索可能な状態で保存される
- クラウド上に7年間以上保存される

つまり、**クラウド会計ソフトに請求書やレシートを取り込むだけで、法令対応が完了する。** 専用の電帳法対応システムを別途導入する必要はない。


## 導入スケジュールの目安

| 期間 | やること | 所要時間 |
|------|---------|---------|

**重要:** 並行稼働期間を設けることで、移行リスクを最小限に抑えられる。「いきなり切り替え」はお勧めしない。
