# CLAUDE.mdとは何か

Claude Codeを使い始めると、最初に気づくことがあります。「毎回同じ指示を繰り返している」と。

「TypeScriptで書いて」「テストはVitestで」「コミットメッセージは日本語で」——プロジェクトごとに決まっているルールを、毎回口頭で伝えるのは非効率です。

CLAUDE.mdは、この問題を解決するファイルです。

## CLAUDE.mdの役割

CLAUDE.mdは、Claude Codeがプロジェクトで作業を始める際に自動的に読み込む設定ファイルです。プロジェクトのルート（またはサブディレクトリ）に配置すると、Claude Codeはその内容を「プロジェクトの前提知識」として理解します。

具体的には、以下の情報を記述できます：

- **コーディング規約** — 使用する言語、フレームワーク、スタイルガイド
- **プロジェクト構成** — ディレクトリ構造の説明、重要なファイルの場所
- **ワークフロー** — テスト方法、ビルドコマンド、デプロイ手順
- **禁止事項** — やってはいけない操作、変更してはいけないファイル
- **品質基準** — コードレビューの基準、パフォーマンス要件

## 3つのスコープ

CLAUDE.mdには、配置場所によって3つのスコープがあります。

### 1. プロジェクトCLAUDE.md

プロジェクトのルートディレクトリに配置します。チーム全員で共有するルールを書きます。

```
my-project/
├── CLAUDE.md          ← プロジェクト全体のルール
├── src/
├── tests/
└── package.json
```

このファイルはGitで管理し、チームメンバー全員が同じルールでClaude Codeを使えるようにします。

### 2. ユーザーCLAUDE.md

`~/.claude/CLAUDE.md` に配置します。個人の好みや、すべてのプロジェクトに適用したい設定を書きます。

```
~/.claude/
└── CLAUDE.md          ← 全プロジェクト共通の個人設定
```

例えば「コミットメッセージは日本語で」「レスポンスは簡潔に」といった個人設定はここに書きます。

### 3. サブディレクトリCLAUDE.md

特定のディレクトリ内だけに適用するルールを書けます。

```
my-project/
├── CLAUDE.md
├── frontend/
│   └── CLAUDE.md      ← フロントエンド固有のルール
└── backend/
    └── CLAUDE.md      ← バックエンド固有のルール
```

モノレポ構成で、フロントエンドとバックエンドで異なるルールを適用したい場合に有効です。

## 最小構成から始める

いきなり大きなCLAUDE.mdを書く必要はありません。以下の3行から始めましょう。

```markdown
# プロジェクトルール

- TypeScriptで書く。型は明示する
- テストはVitestで書く。`npm test` で実行
- コミットメッセージは日本語で、変更の理由を書く
```

これだけで、Claude Codeは毎回これらのルールに従ってくれます。

## CLAUDE.mdが効果的な理由

CLAUDE.mdが単なるREADMEと異なるのは、**Claude Codeが会話開始時に必ず読み込む**点です。プロンプトの一部として取り込まれるため、「このルールに従って」と毎回言う必要がなくなります。

これは、新しいチームメンバーに「このプロジェクトのルールブック」を渡すのと同じです。ただし、AIは渡されたルールを忘れません。
