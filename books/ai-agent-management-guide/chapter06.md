# 営業部門の構築 -- CSOエージェントでリードを獲得する

## 技術者CEOが最も苦手とする部門

正直に言おう。筆者にとって営業は最も苦手な領域だった。

コードを書くのは得意だ。記事を書くのもできる。しかし「見込み客にメールを送る」「電話をかける」「提案書を作る」となると、途端に手が止まる。技術者出身のCEOには共感してもらえるのではないだろうか。

しかし、ビジネスは売上がなければ続かない。マーケティングでトラフィックを獲得しても、最終的に「この人に買ってもらう」というアクションがなければ収益は生まれない。

AIエージェントは、この営業のハードルを劇的に下げてくれる。リスト作成、HP分析、提案書生成、メール文面の作成。営業プロセスの8割は「調査と文書作成」であり、これはまさにAIの得意領域だ。残りの2割、つまり「人間関係の構築」と「最終的な意思決定への働きかけ」だけを人間が担えばよい。


## 営業部門エージェントの全体設計

### ディレクトリ構成

```
.company/
  departments/
    sales/
      STATE.md                    # 営業部門の現在状態
      hp-renewal/                 # HP制作・リニューアル事業
        prospect-list.md          # 見込み客リスト
        templates/
          outreach-email.md       # メールテンプレート
          phone-script.md         # 電話スクリプト
          proposal-template.md    # 提案書テンプレート
          pricing.md              # 料金表
        outreach/                 # 個別メール下書き
          {会社名}-email.md
        proposals/                # 個別提案書
          {会社名}-proposal.md
        reports/                  # 分析レポート
          {会社名}.md
        site-analyses.json        # サイト分析データ
```

### CSOエージェントの定義

```markdown
# 営業部門エージェント（CSO）

## 役割
リード獲得から提案・クロージングまでの営業プロセスを統括する。
対外的なアクション（メール送信、電話）は必ずdraftモードで生成する。

## タスク分類

| カテゴリ | 内容 | 権限レベル |
|---------|------|-----------|
| リスト作成 | 業種・地域で見込み客を調査 | execute |
| HP分析 | 見込み客のWebサイトを分析 | execute |
| 提案書生成 | 分析結果をもとに提案書を作成 | draft |
| メール作成 | 個別カスタマイズしたアウトリーチメール | draft |
| パイプライン管理 | 商談状態の更新・レポート | execute |
| 見積もり作成 | 案件の見積書を生成 | draft |

## 重要ルール
- メール送信は「絶対に」自動実行しない。必ずCEO承認を経る
- 提案書の数字（金額、納期）はCEOが最終確認する
- 顧客情報はGit管理し、変更履歴を追跡する
```


## HP分析から提案書自動生成の仕組み

筆者の会社では、HP制作・リニューアルの営業にエージェントを活用している。この事業は営業エージェントとの親和性が非常に高い。理由は、**見込み客のHPという「分析対象」が公開情報として誰でもアクセスできる**からだ。

### 分析フロー

```
1. リスト作成
   - 業種（歯科医院、税理士事務所など）と地域を指定
   - エージェントが公開情報からリストを生成

2. HP分析
   - 各社のWebサイトにアクセス
   - 以下の観点で分析:
     - モバイル対応状況
     - ページ表示速度
     - SEOの基本要素（title、meta description、h1）
     - デザインの印象（古い/モダン）
     - コンテンツの充実度
     - 予約導線の有無
   - 分析結果をJSON形式で保存

3. 提案書生成
   - テンプレートに分析結果を流し込む
   - 具体的な改善ポイントを列挙
   - 概算見積もりを添付

4. メール作成
   - テンプレートをベースに個別カスタマイズ
   - 分析で見つけた具体的な課題に言及
   - 無料相談のCTAを配置
```

### テンプレート駆動の提案書

テンプレートを用意しておくことで、エージェントの出力品質が安定する。

```markdown
# 提案書テンプレート

## {会社名} 様 ホームページリニューアルのご提案

### 現状分析

#### サイトの良い点
- {good_point_1}
- {good_point_2}

#### 改善が見込まれる点
- {issue_1}: {具体的な説明}
- {issue_2}: {具体的な説明}
- {issue_3}: {具体的な説明}

### ご提案内容

| 項目 | 内容 | 効果 |
|------|------|------|
| {item_1} | {description} | {expected_effect} |
| {item_2} | {description} | {expected_effect} |

### お見積もり

| プラン | 内容 | 金額（税込） |
|--------|------|-------------|
| ライトプラン | {内容} | {金額} |
| スタンダードプラン | {内容} | {金額} |
| プレミアムプラン | {内容} | {金額} |

### 制作の流れ
1. ヒアリング（1週間）
2. デザイン案作成（2週間）
3. 実装・テスト（3週間）
4. 公開・運用開始
```

このテンプレートにエージェントが分析データを埋め込むことで、個別にカスタマイズされた提案書が自動生成される。

### メールテンプレートの設計

アウトリーチメールも同様にテンプレート化する。

```markdown
# メールテンプレート

件名: {会社名}様のホームページについて（改善のご提案）

{担当者名} 様

突然のご連絡失礼いたします。
合同会社ジョインクラスの{CEO名}と申します。

{業種}のホームページ制作を専門としており、
{地域}の{業種}様のサイトを拝見する中で、
{会社名}様のホームページについて気づいた点がございましたので
ご連絡させていただきました。

{具体的な課題の指摘 - 1-2文}

もしよろしければ、無料でサイトの改善レポートをお送りいたします。
ご興味がおありでしたら、本メールにご返信いただけますと幸いです。

合同会社ジョインクラス
{CEO名}
```

ポイントは**具体性**だ。「御社のホームページを改善しませんか」という一般的なメールは無視される。「御社のスマートフォン表示でメニューが崩れている点」のように、**その会社固有の課題を指摘する**ことで開封率と返信率が大きく変わる。


## 承認パイプライン: 対外アクションの安全管理

営業部門は、対外的なアクションが多い部門だ。メールの誤送信、不適切な金額提示、事実と異なる説明。いずれも会社の信用に直結するリスクがある。

そのため、営業部門では**すべての対外アクションを承認パイプラインに通す**ことを絶対ルールにしている。

```markdown
# 営業部門の承認フロー

## 承認必須（draft）
- アウトリーチメール
- 提案書
- 見積書
- 電話スクリプト

## 自動実行（execute）
- 見込み客リストの作成
- HP分析
- パイプライン状態の更新
- 分析レポートの生成

## 承認時のチェックポイント
- [ ] 会社名・担当者名が正しいか
- [ ] 金額に誤りがないか
- [ ] 事実と異なる記述がないか
- [ ] 送信先メールアドレスが正しいか
- [ ] トーンが適切か（押し売り感がないか）
```

承認にかかる時間は1件あたり2-3分だ。10件のメールを承認しても30分で済む。エージェントが下書きを作成する時間と比較すれば、CEOの作業負担は極めて小さい。


## 実践例: 20社へのアウトリーチを1日で準備する

筆者が実際に行った営業キャンペーンの例を紹介する。

### 背景

HP制作・リニューアル事業の立ち上げにあたり、地域の歯科医院・税理士事務所・整骨院にアウトリーチを行う必要があった。

### 作業の流れ

```
09:00 /ai-ceo:sales:pipeline "さいたま市 歯科医院 HP古い"
      → エージェントが20社のリストを生成
      → 各社のHP分析を自動実行
      → 所要時間: 約30分

09:30 分析結果の確認
      → site-analyses.json に分析データが蓄積
      → 特に課題が多い10社を優先ターゲットに選定

10:00 /ai-ceo:sales:proposal "歯科医院10社"
      → 10社分の提案書を自動生成
      → 所要時間: 約20分

10:20 /ai-ceo:sales:outreach "歯科医院10社"
      → 10社分のメール下書きを生成
      → 各メールに個別の課題指摘を盛り込み
      → 所要時間: 約15分

10:35 CEO承認作業
      → 10通のメール + 10件の提案書を確認
      → 金額、会社名、課題指摘の正確性を検証
      → 2件の修正指示 → エージェントが即時修正
      → 所要時間: 約40分

11:15 準備完了
      → 承認済みメール10通 + 提案書10件
```

結果として、午前中の約2時間で10社分の営業準備が完了した。人間が1社ずつHP分析→提案書作成→メール文面作成をやると、1社あたり1-2時間かかる。10社で10-20時間だ。エージェントはこれを約2時間に短縮してくれた。

### 成果

10社にメールを送信した結果:
- 返信: 3社（返信率30%）
- うち面談: 2社
- うち受注: 1社

返信率30%は、コールドメールとしては高い水準だ。これは各メールで**その会社固有の課題を具体的に指摘した**ことが大きい。「テンプレートの一斉送信」ではなく「個別分析に基づく提案」というアプローチが、AIエージェントだからこそ可能になる。


## パイプライン管理

商談の進捗は `prospect-list.md` で管理する。

```markdown
# 見込み客パイプライン

| 会社名 | 業種 | ステータス | 次アクション | 期日 |
|--------|------|-----------|------------|------|
| A歯科 | 歯科 | 面談済み | 見積書送付 | 3/7 |
| B税理士 | 税理士 | メール返信あり | 面談日程調整 | 3/5 |
| C整骨院 | 整骨院 | メール送信済 | フォローアップ | 3/10 |
```

`/ai-ceo:morning` の朝ダイジェストで、営業パイプラインの状態も確認できる。期日が近いアクションがあれば、エージェントが自動的にリマインドしてくれる。


## 営業部門構築のチェックリスト

- [ ] CSOエージェントの定義ファイルを作成した
- [ ] 提案書テンプレートを準備した
- [ ] メールテンプレートを準備した
- [ ] 承認パイプラインを設定した（対外アクションはすべてdraft）
- [ ] パイプライン管理の形式を決めた
- [ ] 最初の営業キャンペーンを実行した
- [ ] 返信率・受注率を記録し、改善サイクルを開始した

営業は「質 x 量」のゲームだ。AIエージェントが「量」を担保し、人間が「質」（関係構築、最終交渉）を担う。この分業が、ひとり会社の営業を実現可能にする。
