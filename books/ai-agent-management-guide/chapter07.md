# 経理・財務部門の構築 -- CFOエージェントで数字を管理する

## 数字を把握していないCEOは経営していない

ひとり会社のCEOにとって、経理は「やらなければいけないがやりたくない」業務の筆頭だろう。領収書の整理、仕訳の入力、月次決算、確定申告。どれも地味で時間がかかり、しかもミスが許されない。

しかし、経理を疎かにするとビジネスは確実に破綻する。「今月いくら使って、いくら入ってきて、手元にいくらあるのか」を把握していないCEOは、経営しているのではなく漂流しているだけだ。

筆者も創業当初は経理を後回しにしていた。確定申告の直前に1年分のレシートを引っ張り出して徹夜で仕訳を入力する、という最悪のパターンだ。この反省から、CFOエージェントの構築に本腰を入れた。

結果として、月次のコスト管理が自動化され、リアルタイムに近い経営数字の把握が可能になった。ただし、**経理は最も「人間の最終確認」が重要な部門**でもある。数字の間違いは税務署との問題に直結するからだ。


## 経理部門エージェントの全体設計

### ディレクトリ構成

```
.company/
  departments/
    finance/
      STATE.md                  # 経理部門の現在状態
      cost-tracking.md          # コスト管理台帳
      monthly-reports/          # 月次レポート
        2026-01.md
        2026-02.md
      invoices/                 # 請求書管理
        {client}-{date}.md
      tax/                      # 税務関連
        calendar.md             # 税務カレンダー
        deductions.md           # 経費・控除リスト
```

### CFOエージェントの定義

```markdown
# 経理・財務部門エージェント（CFO）

## 役割
会社の財務状態を正確に把握し、経営判断に必要な数字を提供する。
数字の正確性が最優先。不明確な場合はCEOに確認を求める。

## タスク分類

| カテゴリ | 内容 | 権限レベル |
|---------|------|-----------|
| コスト記録 | 日々の支出をcost-tracking.mdに記録 | execute |
| 月次レポート | 月次の収支サマリーを生成 | execute |
| コスト分析 | カテゴリ別支出分析、前月比較 | read-only |
| 請求書作成 | クライアントへの請求書ドラフト | draft |
| 収支予測 | 向こう3ヶ月の収支予測 | read-only |
| 税務準備 | 仕訳チェック、確定申告準備 | draft |

## 重要ルール
- 金額の推測をしない。不明な場合はCEOに確認する
- 税務判断をAIだけで行わない。必ず人間が最終確認する
- すべての数字の根拠（ソース）を明記する
```

### なぜMarkdownで管理するのか

「会計ソフトを使えばいいのでは」という疑問があるだろう。もちろん正式な帳簿は会計ソフト（freee、マネーフォワードなど）で管理する。`cost-tracking.md` はそれとは別の、**経営判断のための速報値**だ。

Markdownで管理する利点は以下の3つ。

1. **AIエージェントが直接読み書きできる**: 会計ソフトのAPIを叩くよりも、テキストファイルの操作のほうが圧倒的にシンプル
2. **Git管理で変更履歴が追える**: いつ、何が変更されたかが一目瞭然
3. **他の部門ファイルと同じ形式**: STATE.mdや承認キューと同じ仕組みで扱える

正確な帳簿は会計ソフト、速度優先の経営数字はMarkdown。この二層構造がひとり会社には合っている。

## 税理士エージェントとの連携

AI-CEO Frameworkでは、CFOエージェントとは別に「税理士エージェント」を用意している。経理の日常業務と、税務の専門判断を分離するためだ。

### 税理士エージェントの役割

```markdown
# 税理士エージェント

## 役割
税務に関する専門的な判断支援を行う。
最終判断は必ず人間（CEO or 顧問税理士）が行う。

## タスク

| タスク | コマンド | 内容 |
|--------|---------|------|
| 取引データ導入 | /ai-ceo:tax:import | CSVインポート、自動仕訳分類 |
| 仕訳チェック | /ai-ceo:tax:review | 仕訳の正確性確認、異常値検出 |
| 確定申告準備 | /ai-ceo:tax:prep | 決算整理仕訳の洗い出し |
| 節税レビュー | /ai-ceo:tax:save | 利用可能な控除・特例の確認 |
| 税務カレンダー | /ai-ceo:tax:calendar | 申告期限、納付期限の管理 |
```

### 連携フロー

```
日常:
  CFOエージェント → cost-tracking.mdに記録
  ↓
月次:
  CFOエージェント → 月次レポート生成
  税理士エージェント → 仕訳チェック（/ai-ceo:tax:review）
  ↓
確定申告期:
  税理士エージェント → 確定申告準備（/ai-ceo:tax:prep）
  税理士エージェント → 節税レビュー（/ai-ceo:tax:save）
  ↓
CEO → 最終確認 → 申告
```

### 税務カレンダーの管理

ひとり会社で最も危険なのは「申告期限を忘れる」ことだ。税理士エージェントが税務カレンダーを管理し、`/ai-ceo:morning` の朝ダイジェストで期限が近い税務イベントを通知してくれる。

```markdown
# 税務カレンダー 2026年

| 期限 | イベント | ステータス |
|------|---------|-----------|
| 3/15 | 確定申告（所得税） | 準備中 |
| 3/31 | 確定申告（消費税） | 未着手 |
| 6/30 | 住民税（第1期） | 未到来 |
| 8/31 | 個人事業税（第1期） | 未到来 |
```


## 注意点: 会計データの正確性は人間が最終確認

経理部門のエージェント構築で最も強調したいのがこの点だ。

AIは仕訳の「パターン」を学習して自動分類できる。しかし、以下のようなケースでは誤分類のリスクがある。

| ケース | リスク |
|--------|-------|
| 接待交際費 vs 会議費 | 金額と目的で判断が分かれる |
| 固定資産 vs 消耗品費 | 10万円の境界線の判断 |
| 事業経費 vs 私的支出 | 按分の判断が必要 |
| 売上の計上時期 | 発生主義 vs 現金主義 |

これらの判断を誤ると、税務調査で指摘を受けるリスクがある。エージェントが「これは交際費だと思います」と分類しても、**最終的な判断は人間が行う**べきだ。

筆者の運用ルールは以下の通り。

```markdown
## CFOエージェントの運用ルール

1. エージェントが仕訳を自動分類する
2. 分類に自信がない項目は「要確認」フラグを付ける
3. CEOが週1回「要確認」項目をレビューする
4. 四半期ごとに全仕訳を一括レビューする
5. 確定申告前に顧問税理士（いれば）に最終チェックを依頼する
```

AIに任せすぎず、人間の目を通す仕組みをルールに組み込んでおくことが重要だ。
