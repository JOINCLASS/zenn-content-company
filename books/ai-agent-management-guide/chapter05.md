# マーケティング部門の構築 -- CMOエージェントで集客を自動化する

## プロダクトを作っただけでは売れない

前章で開発部門のエージェントを構築した。プロダクトが安定して回る体制ができた。しかし、ここで多くの技術者CEOが陥る罠がある。「良いものを作れば自然に売れる」という幻想だ。

筆者も同じ過ちを犯した。プロダクトの品質に自信があったが、リリース後1ヶ月のユーザー数はゼロに近かった。技術ブログも書いていたが、SEOを意識しない記事は検索流入がほぼ発生しない。SNSで告知しても、フォロワーが少なければリーチは限定的だ。

この経験から、マーケティング部門は開発部門と同じレベルで構築すべきだと確信した。そしてAIエージェントは、マーケティングの「量」の問題を解決してくれる。SEO記事を週3本書く、SNSを毎日投稿する、Zenn書籍を月1冊出す。人間1人では不可能な量産を、エージェントが可能にする。


## マーケティング部門エージェントの全体設計

### ディレクトリ構成

```
.company/
  departments/
    marketing/
      STATE.md                    # マーケ部門の現在状態
      sns-content-calendar.md     # SNSコンテンツカレンダー
      books/                      # Zenn書籍管理
        {book-slug}/
          config.yaml
          chapter-01.md
          ...
      drafts/                     # 下書き管理
        {platform}-{topic}.md
  steering/
    brand.md                      # ブランドガイドライン
```

### CMOエージェントの定義

```markdown
# マーケティング部門エージェント（CMO）

## 役割
トラフィック獲得とブランド構築の統括。
「読まれるだけのコンテンツ」ではなく「行動を起こさせるコンテンツ」を作る。

## タスク分類

| カテゴリ | 例 | 権限レベル | 品質基準 |
|---------|-----|-----------|---------|
| SEO記事作成 | Zenn技術記事 | draft | SEOスコア80+、CTA配置 |
| SNS投稿 | X、LinkedIn投稿 | draft | エンゲージメント設計 |
| 書籍執筆 | Zenn Books | draft | 章構成、無料プレビュー設計 |
| LP作成 | プロダクトページ | draft | CVR最適化 |
| コンテンツ企画 | 月間カレンダー | execute | トレンド分析 |
| 分析レポート | PV、CVRレポート | read-only | データ正確性 |

## 収益導線の設計

すべてのコンテンツに以下の導線を組み込む:
1. Zenn記事 → 記事内CTA → LP → ユーザー登録
2. Zenn書籍 → 書籍販売収益 + ブランディング
3. SNS → フォロワー獲得 → LP訪問 → 登録
4. LP → 訪問 → 無料トライアル → 有料転換
```


## コンテンツ戦略の設計

### SEO記事の量産体制

マーケティングにおいてSEO記事は「資産」だ。一度書けば、検索流入が継続的に発生する。エージェントを使えば、この資産を効率的に積み上げられる。

筆者が実践しているSEO記事の制作フローは以下の通りだ。

```
1. キーワード選定
   - 検索ボリュームと競合度を分析
   - 自社プロダクトとの関連性を評価
   - 検索意図を分類（情報型 / 商取引型 / ナビゲーション型）

2. 構成案の作成
   - H2/H3の見出し構成
   - 各セクションの要点
   - CTA配置ポイントの決定

3. 記事執筆（エージェント）
   - 2000-4000字
   - 実体験・実データの盛り込み（E-E-A-T対策）
   - 自社プロダクトへの自然なCTA配置
   - 関連記事への内部リンク

4. Zenn CLIフォーマットで出力
   - frontmatter最適化（title, emoji, type, topics）
   - Markdown構造の最適化
```

### キーワード選定の実例

筆者の場合、以下のような視点でキーワードを選定している。

| キーワード例 | 検索意図 | プロダクト関連度 | 記事方針 |
|------------|---------|----------------|---------|
| 「Claude Code 使い方」 | 情報型 | 高 | チュートリアル + ツール紹介 |
| 「AIエージェント 経営」 | 情報型 | 最高 | 実践事例 + 書籍・コンサルCTA |
| 「ひとり社長 効率化」 | 商取引型 | 中 | 課題共感 + プロダクトCTA |
| 「Zenn 技術記事 書き方」 | 情報型 | 低 | 認知拡大目的 |

ポイントは「プロダクト関連度が高いキーワード」を優先することだ。PVが多くても、自社のビジネスと無関係なキーワードでは売上につながらない。


## コンテンツカレンダーの自動生成

週単位でコンテンツの公開スケジュールを管理する。エージェントが自動でカレンダーを生成し、CEOが承認する流れだ。

```markdown
# 週間コンテンツカレンダー（3月第1週）

| 曜日 | プラットフォーム | コンテンツ | ステータス |
|------|----------------|-----------|-----------|
| 月 | Zenn | 「Claude Codeの実践Tips 10選」 | 下書き完了 |
| 火 | X | 記事告知ツイート（3パターン） | 下書き完了 |
| 水 | LinkedIn | AIエージェント経営の実践報告 | 下書き完了 |
| 木 | Zenn | 「CLAUDE.md設計パターン集」 | 執筆中 |
| 金 | X | 書籍宣伝ツイート | 下書き完了 |
```

このカレンダーを `/ai-ceo:mkt:content-plan` コマンドで自動生成し、`sns-content-calendar.md` に保存する。


## Zenn書籍の執筆戦略

Zenn Booksは、ひとり会社にとって理想的な収益チャネルだ。

### なぜZenn Booksか

- **販売手数料が低い**: 売上の86.8%が手元に残る（Zennの取り分は13.2%）
- **技術者へのリーチ**: Zennのユーザー層は技術に関心が高く、自社プロダクトの潜在ユーザーと一致する
- **SEO効果**: 書籍もZenn内検索・Google検索で発見される
- **ブランディング**: 書籍著者としての権威性がコンサルティング案件につながる

### 書籍制作のエージェント活用

```
1. 企画立案
   - テーマ選定: 自分の専門性 × 市場の需要
   - ターゲット設定: 誰の、何の課題を解決するか
   - 価格設定: 500-2000円（内容の深さに応じて）
   - 章構成: 10-15章、各章3000-5000字

2. 執筆（エージェント活用）
   - 各章のアウトラインを人間が作成
   - エージェントが初稿を生成
   - 人間が実体験・具体例を追加して編集
   - エージェントが構成・表現を最適化

3. 公開設定
   - 無料プレビュー: 序章 + 最初の2-3章を無料公開
   - 購入動機の設計: 無料章で「もっと知りたい」を作る
```

重要なのは、**エージェントに丸投げしない**ことだ。AIが生成した文章はそのままでは「どこかで読んだことがある」印象になりがちだ。自分の実体験、具体的な数字、失敗談を追加することで、読者の信頼を獲得できる。これはE-E-A-T（Experience, Expertise, Authoritativeness, Trustworthiness）の観点からもSEOに有効だ。


## SNS投稿の設計

### X（Twitter）の投稿パターン

1つのコンテンツに対して、3つのパターンで投稿を用意する。

```markdown
## パターン1: 告知型
「Claude Codeで会社の経営を自動化している話を書きました。
月5万円で6プロダクトを同時運営できる仕組みの全体像。」
→ 記事リンク

## パターン2: 学び共有型
「AIエージェント経営を3ヶ月やってわかったこと:
- 承認パイプラインがないと事故る
- CLAUDE.mdは細かく書くほど品質が上がる
- 最初は全部門同時に作らない
詳しくはこちら」→ リンク

## パターン3: 質問型
「ひとり社長で月に何時間コードを書いていますか？
自分は3ヶ月前まで月100時間でしたが、
AIエージェント導入後は月20時間になりました。
浮いた80時間で何をしているかを書きました」→ リンク
```

パターンを複数用意する理由は、どの切り口がエンゲージメントを獲得しやすいかをテストするためだ。反応が良いパターンを分析して、次回以降の投稿に活かす。

### LinkedIn投稿の設計

LinkedInはXとは異なるユーザー層にリーチできる。ビジネスオーナーや経営者が多いため、「経営効率化」「コスト削減」の切り口が刺さる。

```markdown
## LinkedIn投稿テンプレート

### 冒頭（フック）
1行目で「おっ」と思わせる。
例: 「社員ゼロ、売上は右肩上がり。その秘密は...」

### 本文（3-5段落）
- 課題の提示
- 自社の取り組み
- 具体的な成果（数字）
- 学び

### CTA
- 記事や書籍へのリンク
- 「同じ課題を持つ方はDMください」
```


## 品質基準の設計

マーケティングコンテンツの品質が安定しないと、ブランドイメージが毀損する。以下の基準をエージェントの定義に組み込んでおく。

```markdown
# マーケティング品質基準

## 全コンテンツ共通
- [ ] CTA（プロダクトへの導線）が含まれているか
- [ ] ブランドガイドライン（brand.md）に準拠しているか
- [ ] 具体的な数字や実例が含まれているか
- [ ] 誤字・脱字がないか

## SEO記事
- [ ] タイトルにメインキーワードが含まれているか
- [ ] H2/H3が論理的に構成されているか
- [ ] メタディスクリプションが適切か（120-160文字）
- [ ] 内部リンクが2つ以上含まれているか

## SNS投稿
- [ ] プラットフォームの文字数制限を守っているか
- [ ] ハッシュタグが適切か（3-5個）
- [ ] エンゲージメントを促す要素があるか（質問、CTA）

## Zenn書籍
- [ ] 無料プレビュー章で購入動機が作れているか
- [ ] 各章が独立して読めるか
- [ ] 実践的なコード例やテンプレートが含まれているか
```


## 実践例: Zenn記事のネタ出しから公開まで

マーケ部門エージェントを使った記事制作の実際のフローを示す。

### Step 1: ネタ出し（エージェント）

```
/ai-ceo:mkt:content-plan

→ エージェント出力:
今週の記事候補:
1. 「CLAUDE.mdの書き方完全ガイド」（検索需要: 高）
2. 「Claude Codeでテストを自動生成する方法」（検索需要: 中）
3. 「AIエージェントで確定申告を効率化した話」（検索需要: 中、話題性: 高）
```

### Step 2: 構成案の作成（エージェント）

CEOが「1番で」と指示すると、エージェントが構成案を生成する。

### Step 3: 記事執筆（エージェント + 人間）

エージェントが初稿を生成し、CEOが実体験を追加・編集する。

### Step 4: 承認・公開

完成した記事が `approval-queue.md` に追加される。CEOが内容を確認し、承認すれば公開される。

この一連のフローで、1記事あたりの作業時間は約30分だ。エージェントなしでゼロから書くと2-3時間かかることを考えると、大幅な効率化になる。
